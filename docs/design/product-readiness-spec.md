# Продуктовая готовность (без контента) — спека волны

Грунт: RS-01 (методика — здоровая геймификация БЕЗ streak-shaming, desirable difficulty,
spacing, калибровка) + locked-дизайн (крем #F6F1E9 + коралл #F0533A + Rubik + спарк-знак).
Цель: приложение ощущается ЗАВЕРШЁННЫМ продуктом, а не «просто работает». Контент уроков НЕ трогаем.

## 1. Петля ежедневного возврата (здоровая, без наказания)
- **Home = «сессия на сегодня»**: явный CTA «Продолжить сегодня» с числом карт к повтору + оценкой минут.
  Первый экран за ~10 сек даёт очевидное вознаграждающее действие (сессия или свежий урок) — нет тупиков.
- **Стрик**: показываем как поддержку; НЕ стыдим за пропуск (research: streak-shaming вреден). Разрыв —
  тихо, с тёплым «начни новую серию», без красного/вины. Празднуем вехи (3/7/30 дней) — сдержанно.
- **XP + уровень**: видимый прогресс; на верном ответе — маленькая награда (XP-bump + хаптика).
- **Экран «день закрыт»**: после прохождения всех due-карт — удовлетворяющее завершение (XP за сессию,
  стрик +1, превью «завтра: N карт»), мягкий «возвращайся завтра». Закрытие с forward-hook, не пустота.
- **Мета-прозрачность** (методика): где уместно — честно показать, что расписание = FSRS (учебный экспонат).

## 2. Edge / empty / first-run / error / loading
- **First-run (новый юзер)**: лёгкий тёплый онбординг — что это (1 экран/инлайн-герой), «начни с value-types»,
  без аккаунта. Не мешает опытному — можно сразу в дело.
- **Empty-состояния** (все явные, тёплые, с действием):
  - нет due + есть непройденные уроки → «нет карт к повтору · пройди новый урок» + CTA урока.
  - нет due + все уроки пройдены → «всё повторено на сегодня · вернись завтра» + превью след. повтора.
  - все уроки пройдены → поздравление + «повторяй, чтобы закрепить».
- **Loading**: скелетоны (не спиннер/пустота) на home/progress/profile во время загрузки.
- **Error + retry**: КАЖДЫЙ сетевой вызов → дружелюбная ошибка + кнопка «Повторить» (кейс — офлайн в дороге).
  Единый error-компонент. Никаких «сырых» 401/500 на экране.

## 3. Общий UX-полиш
- **Переходы**: плавные между home↔урок↔progress↔profile; уважают reduced-motion.
- **Хаптика** (Telegram): согласованно на ключевых действиях — верно/неверно (notification success/error),
  выбор оценки (impact), смена таба (selection), завершение урока (success). Не спамить.
- **Консистентность**: единые отступы/типографика/состояния по всем экранам; одинаковые кнопки, заголовки,
  паттерны empty/loading/error; a11y (focus-visible, контраст) сохранены; safe-area/stable-viewport целы.
- **Микро-интеракции**: XP-bump, пламя стрика, заливка progress-ring, press-состояния кнопок — со вкусом,
  в бренде (коралл/крем/Rubik), НЕ детсад. Все анимации под reduced-motion.

## Ограничения
