# ТЗ — «Codex»: кино-обучалка фундамента разработки (Telegram Mini App)

<!-- ЯКОРЬ намерения. Язык артефакта — RU; код/идентификаторы — EN. -->

## Цель
Telegram Mini App, куда джун-C#-разработчик заходит каждый день и через научно обоснованное
активное припоминание (движок FSRS-6) глубоко изучает и УДЕРЖИВАЕТ фундамент разработки —
подавая это в киношно-красивой, понятной форме без потери деталей. Успех волны 1: пользователь
проходит ежедневную сессию по вертикальному срезу «ядро C#», карточки реально планируются FSRS
(состояние переживает перезапуск, due-очередь меняется по дням), каждый факт привязан к
первоисточнику, и хотя бы один алгоритм/структура показан живой пошаговой визуализацией —
всё в визуальном языке «Cinematic App Shell».

## Контекст
Джун-C# хочет дойти до экспертного «как будто я сам всё написал» по C#/.NET, CS-фундаменту, БД,
очередям, паттернам, алгоритмам, тому «как работают LLM/Claude/Codex» и тюнингу локальных
моделей — интересно и «не больно». Инсайты: RS-00 (13 углов: двойное ограничение «глубина ×
простая форма» → слои глубины + научная механика), RS-01 (методика: FSRS-6, генерация не
узнавание, таргет ~85% desirable difficulty, калибровка), RS-03 (эталоны: Execute Program =
прямой loop-эталон, Duolingo = juice+анти-чеклист, Anki/FSRS = движок, VisuAlgo = визуализация),
RS-04 (curriculum: 7 треков, ~1980–2480 карточек, срез A рекомендован), RS-05 (риск №1 =
точность контента), RS-07 (арт-дирекшн, выбор пользователя = направление 3), probe (стек JS/TS
подтверждён исполнением). Ниша (RS-03 §1.24): глубина EP + визуал Brilliant/VisuAlgo + удержание
Duolingo без тёмных паттернов + движок FSRS — для C#/.NET/CS/LLM, на русском, в Telegram. Никто
её не закрывает.

## Класс результата
software-UI (Telegram Mini App), интерактивный продукт + контент/образовательный подкомпонент.
Применимы: профминимум (узкий скоуп × 100%, реально работает не display-only, планка из ресёрча)
+ интерактив «глубина контента и пейсинг» (core loop, juice, таймлайн новизны, «минута смерти»)
+ риск-профиль «точность фактов» (educational trust).

## Целевая планка
**ГОТОВЫЙ ПРОДУКТ для вертикального среза (ступень вау ≥3)** — реальное, красивое, ежедневно
используемое приложение, обучающее «ядру C#» на отлично, с архитектурой и контент-пайплайном,
расширяемыми волнами на остальные треки. НЕ прототип-заглушка. «Весь curriculum сразу» (≈12–20
мес, вариант D) — не первая волна; ширина растёт волнами. (Срез — вопрос гейта.)

## Ставка вау
Ось ступени 4 (ЕДИНСТВЕННАЯ): **«как будто я сам написал»** — C#-ЯДРО осваивается ПРОИЗВОДСТВОМ,
не узнаванием. Лестница реконструкции прямо на ядре: predict-output → find-bug → **допиши
недостающий метод** (детерминированно проверяемо на скрытых тестах); мета-урок M0.1 (app
показывает свой реальный FSRS D/S/R твоей карточки) — рефлексивная ВЕРШИНА той же оси, а не
отдельная ставка. Бюджет ставки СОСРЕДОТОЧЕН на одной оси (генерация→владение), не распылён.
Signature-в-скелете: walking skeleton волны 1 ОБЯЗАН содержать реальный FSRS + генеративную петлю
(predict/bug/write) НА C#-ЯДРЕ + M0.1 + ≥1 живую визуализацию — а не плоскую колоду и не мета на
периферии. Созвездие-навигация — НЕ ось, а P1-вау-экран (безопасный срез по cut-list критика).
Риск скуки / потолок новизны: срез A = ~4–6 недель нового материала, дальше due-пачка сжимается →
**контрход**: (а) каденс волны 2 (T3 БД) обязателен к старту на ~4–6 неделе — новизна не рвётся;
(б) вариативность генеративных карточек (одна тема → predict→modify→write→оптимизируй) продлевает
новизну внутри темы; (в) interleaving + таргет ~85% успеха. Эта цена показывается на гейте.

- Кино-shell home (editorial-стиль напр. 3): секции тем + прогресс/мастерство, видимый глазами.
  (Созвездие-навигация concept-DAG — P1-вау-экран, не P0.)
- Здоровая геймификация: streak со страховкой (freeze), XP, мастерство узла — БЕЗ тёмных паттернов.
- Ежедневная петля: due-today очередь, дневной лимит новых (форсирует возврат), кино end-card.

## Паспорт вкуса
**Референс-эталон (loop)**: Execute Program — деконструкция RS-03 §2.1;
**визуальный эталон**: направление 3 «Cinematic App Shell» — `docs/research/refs/boards/board-3-cinematic.jpg`
+ `docs/research/refs/cinematic/` (NOCTURNE focus-mode, Scorsese-films editorial, дарк-gallery);
визуализации — `board-2-fui.jpg` (рестрейн); навигация — `board-1-constellation.jpg`.
Builder смотрит на них ДО каждой UI-фичи; evaluator сверяется на майлстоунах (ориентир, не пиксель).

**Деконструкция core loop (из RS-03 §2.1, EP):**
1. core loop: показать пример/вопрос → ученик ПРИПОМИНАЕТ/ГЕНЕРИРУЕТ (предсказать вывод / найти
   баг / прочитать код / объяснить) → фидбек+грейд → FSRS планирует → следующая. Не узнавание.
2. таймлайн первой сессии по минутам: **0–1 мин** кино cold-open (NOCTURNE) → первая карточка,
   реальный recall за ~60с («я реально понял»); **1–3 мин** 2–3 карточки + первая живая
   визуализация + старт streak/XP; **3–5 мин** карточка-генерация + первый узел мастерства
   зажигается на созвездии; **5–10 мин** interleaving старого+новое + карточка-сравнение
   («какой алгоритм где») + кино end-card с итогом и тизером назавтра.
3. частота осмысленного выбора: каждые ~15–30с (ответ/предсказание/градация), не пассивное чтение.
4. кривая эскалации: N+1 требует НОВОГО приёма — predict → modify → write (лестница реконструкции),
   не повтор; interleaving тем растит трансфер.
5. juice-приёмы (≥5, РАЗНЫЕ по событиям, кино-сдержанные): раскрытие ответа = световой sweep+зерно,
   focus-pull; верный recall = тёплый glow-пульс узла + success-haptic; streak = новая звезда на
   созвездии (НЕ «горит-стыди»); мастерство = узел «кристаллизуется» с bloom; шаг визуализации =
   easing+подсветка трассы; «сдался» = приглушённый нейтральный фидбек (иной, чем ошибка); конец
   сессии = кино end-card fade+XP tally. Один эффект на всё = слоп.
6. пороги плотности: ≥5 осмысленных решений за первые 5 минут; ≤3 повтора действия без новизны.
7. таймлайн новизны сессии: каждая сессия = новый концепт + новая визуализация/приём + рост карты;
   «минута смерти» бьётся concept-DAG прогрессией и interleaving. Скоуп среза A покупает ~4–6 недель
   ежедневных сессий нового материала; далее — повторения + следующая волна треков.

**Направление**: cinematic editorial / dark premium (НЕ playful-candy Duolingo, НЕ загромождённый HUD).
**Токены** (в CSS-переменные; хардкод вне токенов запрещён):
- Палитра: `--bg-0:#07070C` (кино near-black), `--bg-1:#101017`, `--bg-2:#191922` (карта),
  `--ink:#ECECF4`, `--muted:#9A9AB2`, `--line:#2A2A38`; `--accent` = ГЕЙТ (cyan `#6BE3FF` /
  amber `#F6B44A` / violet `#9A7CFF` / cyan+amber контраст), `--glow` = bloom акцента.
- Шрифты (НЕ Inter/Roboto/system): дисплей serif **Fraunces**, тело **Geist**, код **JetBrains Mono**.
- Отступы 4/8-кратные; радиусы 4/8/12; зерно-оверлей + bloom на фокус-свечении.
**Анти-слоп запреты**: фиолетовый градиент на белом; Inter; равномерно-робкая палитра; рассыпанные
микроанимации; загромождённый Envato-HUD; детская candy-эстетика; один фидбек на все события;
эмодзи вместо иконок; streak-shaming; guilt-нотификации; монетизация через наказание (energy/hearts).
**Signature-элемент**: кино-момент петли — **focus-pull раскрытие карточки + «кристаллизация» узла
при мастерстве** (в walking skeleton, на оси генерация→владение). Вся смелость сюда; остальное тихо.
Созвездие — P1-вау-экран, не signature волны 1.
**Стек UI**: статический TS TMA (Vite) + @telegram-apps SDK; визуализации на canvas/SVG (лёгкие,
bundle-бюджет); дискретные токены вместо свободного CSS. (Финальный стек-ADR — фаза 3, со спайком.)

## Риски и допущения
- Точность контента — риск №1 → каждый факт из первоисточника + verification-pass (G3), volatile-треки датируются.
- Взрыв scope → срез A волной 1, расширяемая архитектура (ADR контент-модели).
- Ось «как будто я сам написал» для C#-ЯДРА обеспечивает ГЕНЕРАЦИЯ/реконструкция (predict/bug/write на
  скрытых тестах), а НЕ «self-hosting» C#-кода app (фронт на TS; мета M0.1 показывает реальный TS-движок;
  C# живёт в КОНТЕНТЕ). Исполняемый C#-раннер в браузере — позже (P2, нужен доп. спайк).
- Потолок новизны ~4–6 недель (срез A) → каденс волны 2 (T3 БД) обязателен к старту на ~4–6 неделе;
  между волнами дневную ценность держат FSRS-повторения + interleaving + вариативность генеративных карт.
- Пейсинг (длина сессии, доза новых атомов) — ЭМПИРИЧЕСКИЙ гейт (sim-harness), не константа.
- Допущение: принял срез A + стек JS/TS + client-only + только тёмная тема; если неверно — меняется
  объём волны 1 / стек-ADR / добавляется бэкенд.
