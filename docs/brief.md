## Цель
Telegram Mini App, куда СЕНЬОР-разработчик C# заходит регулярно и заново, но ГЛУБОКО, проходит
фундамент (C#/.NET, CS-ядро, БД, очереди, паттерны, как работают LLM, тюнинг моделей) — с
интерактивными АНИМАЦИЯМИ и научной методикой запоминания, доходя до понимания «как будто я сам
спроектировал технологию» (машинный уровень, IL, внутренности, все нюансы). Успех волны 1: реальный
ежедневный loop на FSRS (сервер) + несколько уроков ЭКСПЕРТНОЙ плотности (много анимированных разборов,
каждый объяснён и заземлён на первоисточник), работающие как продукт в Telegram.

## Контекст (что уже решено и построено)
Пройден большой цикл. Зафиксировано: аудитория = СЕНЬОР (рефреш+глубина, не обучение с нуля); дизайн =
**mid** (крем+коралл+Rubik, «ооо это круто»); формат = **lesson-as-data + segments[]** (RS-10,
lesson-format.md); движок анимаций = **living-diagram** (RS-08, спайк 1.28KB, PASS); методика = RS-09
(7-фазный шаблон, expertise-reversal, refutation мисконцепций); эталон плотности доказан = урок boxing
(7 анимир. разборов + IL); построен **бэкенд** (C#/ASP.NET Core + SQLite + FSRS-6 + initData-auth,
запущен, проверен curl). Ставка вау уточнена пользователем: «как будто я сам написал» = про ГЛУБИНУ
ПОНИМАНИЯ технологий, НЕ про написание кода в аппе и не про self-hosting.

## Класс результата
software-UI (Telegram Mini App) + образовательный контент, интерактивный продукт. Аудитория —
СЕНЬОР (калибровка вверх: нюансы/машинный уровень/gotchas, fade лесов). Риск-профиль «точность фактов».

## Целевая планка
**ГОТОВЫЙ вертикальный срез (ступень вау ≥3)**: реально работающее ежедневное приложение с сервером,
на эксперт-глубине. Волна 1 — **тирированная** (RS-13): не 30 тем сразу (~8–11 недель соло), а
**~10–12 уроков** (3–4 флагмана по 7 сегментов + 6–8 стандартных по 2–3), связная вертикаль C#-ядра.
Первый майлстоун — **walking skeleton** (ниже P0-skeleton).

## Язык продукта
`UI language: ru`; код/идентификаторы/IL/термины — English. Allowlist: код-снипеты (C#/SQL/TS),
цитаты первоисточников (MS Learn/arXiv — дословно, с атрибуцией), названия.
Мини-глоссарий: карточка/сегмент/анимация · стек/куча/упаковка · FSRS · мастерство · «Механизм — как и почему».

## Ставка вау (единственная ось)
**«Как будто я сам спроектировал технологию»** — ГЛУБИНА объяснений: каждый концепт разбирается до
внутренностей (IL/память/GC/машина состояний/веса), КАЖДЫЙ нюанс = своя анимация + объяснение
механизма + первоисточник (эталон — boxing: 7 разборов). Не «упомянуть», а показать и объяснить.
Ставочные фичи (кратно больше итераций): эксперт-плотность уроков + аним-движок. Риск скуки у сеньора:
тривиальный вопрос/поверхность → контрход: expertise-reversal (fade поддержки, сразу на нюанс/edge),
refutation мисконцепций, глубина-на-тему вместо ширины. Signature-в-скелете: walking skeleton ОБЯЗАН
содержать ≥1 урок эксперт-плотности (boxing), реально играющий + подключённый к FSRS-петле.

## Скоуп
### P0-SKELETON (первый майлстоун — walking skeleton, чекпойнт пользователю)
- Кино-shell home (mid): секции/прогресс C#-ядра, hero «продолжить», геймификация-со-вкусом.
- **Живой daily-loop через бэкенд**: фронт дёргает `GET /api/due` → урок → `POST /api/review` (FSRS
  двигает расписание); прогресс durable на сервере (SQLite).
- **Telegram-интеграция фронта**: initData → `POST /api/auth`; тема/haptics; работает как реальный TMA.
- ≥2 урока подключены к петле: **boxing (эксперт-плотность, есть)** + ещё 1 (async ИЛИ hashtable),
  анимации автозапускаются при скролле.
- Оффлайн-стратегия под iOS-ограничение (SW не работает в WebView, RS-15): предзагрузка/инлайн уроков
  + кэш прогресса; правда всё равно на сервере.
- Деплой: фронт (GitHub/Cloudflare Pages) + бэк (хост) + бот (BotFather), HTTPS.
### P0 (волна 1 — после скелета)
- Тирированные ~10–12 уроков C#-ядра (флагманы+стандартные) в формате segments[], каждый с
  первоисточником; типы карточек: predict-output, «объясни»(структ. само-объяснение), сравнение,
  cloze, find-bug — БЕЗ обязательного «напиши код» (ставка = понимание, не написание).
- Автоген раскладки виз-сцен из данных (RS-13 рычаг: срезает самую дорогую ручную часть).
- Календарь FSRS, стрик/XP со вкусом, напоминания через бота.
### P1 / P2
- P1: следующие волны-вертикали (T3 БД → T5 паттерны → T4 очереди), калибровка, аналитика прогресса.
- P2: треки T6 LLM / T7 тюнинг; песочница для авторского run-csharp в проде; соц-слой.

## Не входит в скоуп
- Обучение с нуля / азы (аудитория — сеньор). - «Напиши код в аппе, чтобы доказать владение» (отменено
  пользователем). - LLM-судья как гейт оценки (RS-14: bias). - Client-only как источник правды (iOS
  чистит storage — сервер). - 30 тем экспертной плотности в волну 1 (нереально соло).

## Паспорт вкуса
**Референс/эталон**: `docs/research/concepts/mid/index.html` (home) + `concepts/lesson-boxing/index.html`
(эталон урока эксперт-плотности). Токены (см. `design-locked-cream-coral-rubik`): `--bg #F6F1E9`,
`--surface #fff`, `--ink #23201B`, `--muted #8A8377`, `--line #E7DFD2`, `--coral #F0533A`/`--coral-d
#C43D28`, `--sage #7C8B6F`, `--amber #E8A13A` (чуть). Шрифты Rubik(700/800)+Onest+JetBrains Mono.
Радиусы 14/18/22, мягкие тёплые тени. Знак — геометрический «спарк» (не маскот).
**Формат урока**: lesson-as-data (`lesson.segments[]`), каждый сегмент = viz.scenes + «Механизм — как и
почему» + sources; движок living-diagram (SVG→data-join→FLIP→скраббер); **автозапуск при появлении**
(IntersectionObserver, respect prefers-reduced-motion). Плотность: МНОГО анимир. разборов/урок.
**Анти-слоп (mid)**: без золота/тёмного-кино/детсад-зелёного/Inter; без «упоминаний вместо анимаций»;
без streak-shaming; иконки — line-SVG, не эмодзи.
**Signature**: эксперт-плотность (кино-момент петли: focus-pull раскрытие + кристаллизация мастерства).

## Бэкенд (построен, C#/ASP.NET Core)
`backend/Codex.Backend`: SQLite (durable правда о FSRS+прогрессе — фикс iOS-eviction RS-12), FSRS-6
планировщик, initData-HMAC auth, эндпойнты `auth/due/review/lessons` + авторский `run-csharp` (точность
ответа = реальный dotnet-вывод; dev-only, в проде — песочница). Стек C# = учебный артефакт для сеньора.

## Полномочия
Необратимое — только по спросу. Бот-токен создаёт/вводит пользователь (секрет → env, не в git). Деплой
на бесплатный хостинг — подготовлю; публикация в каталог TG — с подтверждения.

## Ограничения
Соло-разработчик + AI-ассист; эксперт-плотность = ~10–14 ч/урок (RS-13) → волна 1 тирирована ~10–12
уроков. iOS: нет service worker в WebView; storage evictable → сервер-правда. Perf-узкое: рантайм анимаций.

## Риски и допущения
- Цена авторинга эксперт-плотности → дрейф к «3 красивым урокам»: контрход — тир + автоген раскладки сцен.
- Точность в масштабе (сотни цитат) → census-аудит G2, не self-pass.
- iOS storage/SW → сервер-правда + предзагрузка (не PWA-SW).
- run-csharp без песочницы (dev) → в проде обязательна изоляция (G10).
- Допущение: волна 1 = C#-ядро вглубь; неверно — сменим фокус на гейте.
