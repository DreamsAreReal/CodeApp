## 3. Находки (утверждение → источник/артефакт → дата → уверенность)

### 3.1 Что FSRS считает «grade»: 4 уровня, бинарный внутри (Q1, фундамент)

- **FSRS требует Rating из 4 значений**: `Again=1`, `Hard=2`, `Good=3`, `Easy=4`. API: `scheduler.review_card(card, rating)` (py-fsrs) / `f.next(card, now, rating)` (ts-fsrs). Библиотека **НЕ поддерживает бинарный pass/fail-вход** — нужно подать один из 4 рейтингов. Источник: py-fsrs README/док (github OSR, fetched 2026-07-09) **[В, API исполнен в спайке]**.
- **Внутри FSRS бинарен по recall**: `Again` = провал припоминания (fail), а `Hard/Good/Easy` = успешное припоминание (pass). Неравенство `Again < Hard < Good < Easy` держится для интервалов. Критическое предупреждение OSR: **нельзя жать «Hard» вместо «Again» при забывании** — FSRS трактует Hard как УСПЕШНОЕ (с усилием) припоминание; путаница ломает модель S. Источник: awesome-fsrs wiki + py-fsrs (github OSR, 2026-07-09) **[В]**.
- **Следствие для генеративных айтемов.** У генеративной карты (в отличие от «сам себе поставил» флеш-карты узнавания) есть **объективный первичный сигнал корректности**: код → G-EXEC (скрытые тесты pass/fail, RS-10 §3.3, исполнено); predict-output → сравнение с реальным stdout; cloze/exact → точное совпадение. Значит «grade» строится НЕ из само-отчёта, а из объективного исхода, модулированного процессными сигналами. Это и есть ответ на Q1 (детали §3.2).

### 3.2 Маппинг генеративного исхода → FSRS Rating (Q1, ядро; спайк исполнен)

**ПРИНЦИП:** корректность — первичная ось (fail→Again безусловно); латентность/подсказки/попытки —
ТОЛЬКО модулятор pass в Hard vs Good; `Easy` НЕ выводится автоматически (это метакогнитивный само-отчёт
«тривиально», иначе искусственно раздуваем стабильность S). Обоснование: RS-01 (recall, не recognition;
generation effect), RS-05 G3 (не вознаграждать пассивный тап — быстрый неверный ответ не должен «пройти»),
desirable difficulty (RS-09 §3.9 — честный Hard кормит FSRS корректно).

Детерминированная функция (спайк, EN identifiers):
```
correct=false                              -> Again   (провал есть провал, усилия не спасают)
correct=true & firstTry & !slow            -> Good     (чистое припоминание — Good, НЕ Easy по умолчанию)
correct=true & (hints>=1 | attempts>=3)    -> Hard     (нужны леса / много попыток — шаткий pass)
correct=true & (attempts==2 | slow)        -> Hard     (одна повторная попытка ИЛИ медленно = Hard)
Easy                                       -> только явный само-отчёт «тривиально»
```
**АРТЕФАКТ-СПАЙК** (`/.spikes/rs14-generative-grading/grade_map.js`, ts-fsrs MIT, исполнено `node`
2026-07-09) — PART 1, вывод из прогона:

| correct | firstTry | hints | attempts | slow | → Rating |
|---|---|---|---|---|---|
| true | true | 0 | 1 | false | **Good** |
| true | true | 0 | 1 | true | **Hard** |
| true | false | 0 | 2 | false | **Hard** |
| true | false | 1 | 2 | false | **Hard** |
| true | false | 0 | 3 | true | **Hard** |
| false | false | 2 | 3 | true | **Again** |

Уверенность: **[В]** для механики (исполнено); пороги (что считать `slow`, `attempts>=3`) — дизайн-ручки,
калибровать метриками (§3.6). Для типов карт по `verify` (RS-10):
- `verify: execute` (predict-output, find-the-bug) — `correct` = stdout совпал с G-EXEC-эталоном;
- `verify: compile` (write-missing-method) — `correct` = скрытые тесты PASS на решении юзера (mutation-проверка силы тестов уже в RS-10 §3.3);
- `verify: exact` (cloze) — точное совпадение;
- `verify: self-graded` (explain) — grade НЕ из этой функции, а из структурированной само-оценки (§3.4).

### 3.3 Expertise-reversal планирование: S как прокси экспертизы, лестница predict→modify→write (Q2)

- **Теория** (наследие RS-09 §3.1): expertise-reversal (Kalyuga et al. 2003, *Educational Psychologist* 38(1):23–31; Kalyuga 2007 обзор *EPR* 19:509–539) — леса, помогающие новичку (worked-example, подсказки), теряют эффект и ВРЕДЯТ (redundancy) с ростом экспертизы. Значит одна карта не оптимальна на всём пути; поддержку надо СНИЖАТЬ по мере мастерства. **[В]**
- **Операционализация RS-14**: прокси текущей экспертизы по атому = **FSRS-stability S** (дни; растёт с усвоением — RS-01 §2.1.4). Правило выбора уровня поддержки на КАЖДОМ due-ревью:
  - `S < S_MODIFY (≈7д)` → **predict** (максимум поддержки: worked-example рядом + predict-the-output/predict-state);
  - `S_MODIFY ≤ S < S_WRITE (≈21д)` → **modify** (faded / find-the-bug / completion — заполни пропуск);
  - `S ≥ S_WRITE (≈21д)` → **write** (минимум поддержки: write-missing-method).
  Пороги — **дизайн-ручки, не константы** (RS-09 открытый Q, требуют sim/AB).
- **АРТЕФАКТ-СПАЙК** (PART 2, тот же файл, исполнено 2026-07-09) — support-level выводится из S и **авто-реинфлируется после лапса**:

| rev# | S(до) | support | исход→Rating | интервал(д) | S(после) |
|---|---|---|---|---|---|
| 1 | 0.00 | predict | pass→Good | 0.01 | 2.31 |
| 2 | 2.31 | predict | pass→Good | 2 | 2.31 |
| 3 | 2.31 | predict | pass→Good | 11 | 10.97 |
| 4 | 10.97 | **modify** | pass→Hard | 32 | 32.23 |
| 5 | 32.23 | **write** | **FAIL→Again** | 0.01 | 2.40 |
| 6 | 2.40 | **predict** ← реинфляция | pass→Good | 2 | 2.40 |
| 7 | 2.40 | predict | pass→Good | 5 | 5.10 |
| 8 | 5.10 | predict | pass→Good | 11 | 10.81 |

**Что доказал спайк:** (а) поддержка автоматически ФЕЙДИТСЯ predict→modify→write по мере роста S
(ревью 1–5); (б) при провале на снятых лесах (write, ревью 5) S обваливается 32→2.40 и support
**сам возвращается к predict** — то есть леса следуют мастерству В ОБЕ стороны, из ЕДИНОГО состояния FSRS,
без отдельной машины состояний. Это конкретная реализация expertise-reversal как правила планировщика.
Уверенность: **[В] для механики (исполнено), [Н] для learning-outcome** — поведенческая модель исхода в
спайке грубая (заглушка), спайк валидирует МЕХАНИКУ связи S↔support, а не пользу для обучения. Польза —
эмпирический гейт (AB на людях, RS-09 §11 Q4).

### 3.4 Автогрейд ПРОЗЫ: почему LLM-судья не годится как источник grade в волне 1 (Q3)

Проза «объясни своими словами» — самый глубинообразующий формат (RS-09 §3.3, Chi 1989/1994: само-объяснение
поднимает recall и transfer), но и самый дорогой для оценки. Рецензируемые данные 2024–2026:

- **LLM–человек согласие лишь умеренное и падает на высоких когнитивных уровнях.** ASAG-обзоры: total-score
  QWK **0.585–0.640** (умеренное); отдельные конфиги 0.67–0.80, но **criterion-level согласие падает с ростом
  когнитивной сложности** — именно там, где живёт «экспертное объяснение». Источник: Emirtekin 2026 (*J. Computer
  Assisted Learning*, Wiley); ASAG-подборка EDM 2025 (2026-07-09) **[В]**.
- **Mid-range degradation (решающий аргумент).** LLM-грейдеры дают **U-образную** точность: на крайних ответах
  (0–1 и 9–10) <1 категориальной ошибки, а на СРЕДНИХ (4–7) — **>2 ошибок против ~0.2 у человека и 0.5 у
  дообученного BERT**. То есть LLM ошибается ровно на ПОГРАНИЧНЫХ ответах — а это ровно те учащиеся, кто
  БОЛЬШЕ всего выигрывает от формативного фидбэка. Источник: «Quality-Conditioned Agreement…» (arXiv 2605.07647,
  2026-07-09) **[В]**. Для нашего SRS это фатально: grade кормит FSRS-интервал, а ошибка на середине — самый
  частый случай у растущего джуна.
- **Over-scoring слабых ответов** (систематический уклон LLM завышать низкокачественные ответы) — прямой путь
  к hypercorrection-риску RS-05 §3.3 (уверенно заученная ошибка дорого лечится). Источник: ASAG-литература выше **[В]**.
- **RS-05 U12 (наследие):** LLM-судья 60–68% согласия с экспертами + **authority bias** (хвалит ответ с
  фейковой ссылкой) + self-preference bias. LLM-судья как ЕДИНСТВЕННЫЙ гейт запрещён. **[В]**

**Вывод по Q3:** LLM-судья НЕ источник grade для FSRS в волне 1. Допустимая роль LLM — **не-блокирующая**:
(1) генерировать наводящие вопросы/хинты; (2) флагать расхождение «само-оценка vs модель-ответ» на ОТЛОЖЕННОЕ
человек-ревью (Stage 4 RS-10, сигнал, не пропуск). Grade прозы в FSRS — из само-оценки (§3.5), не из LLM.

### 3.5 Автогрейд прозы: структурированная само-оценка по ключевым пунктам (рекомендация волны 1) (Q3)

- **Свободная само-оценка умеренно надёжна и улучшается рубрикой/опытом.** Мета-анализ «How Accurate Are Our
  Students?» (Andrade et al., *Educational Psychology Review* 2023, 35: статья; **160 статей, 29 352 участника**,
  9 модераторов): само-оценка коррелирует с оценкой преподавателя **r≈0.59** (умеренно); точность ВЫШЕ при
  использовании **рубрики**, опыте само-оценки и содержательном знании; НИЖЕ у слабых учащихся (склонны
  завышать). Источник: EPR 2023 (link.springer.com, 2026-07-09; число r≈0.59 — из сводки, класс A-мета) **[В/С]**.
- **Механизм волны 1 — structured self-explanation self-grade (не свободный «поставь себе оценку»):**
  1. Учащийся пишет/проговаривает объяснение (self-explanation, Chi — генеративно).
  2. Reveal показывает **N ключевых пунктов модели-ответа** (key-points checklist из `depth.mechanics`/`spec`
     lesson-as-data, RS-10 §5 — уже структурированы как `claim`-ы с источником).
  3. Учащийся отмечает, какие пункты он покрыл → **score = #hit/N** (полу-объективно, рубрика встроена).
  4. Маппинг в grade: `#hit/N` через ту же логику §3.2 (напр. ≥0.8 & без reveal-подсказок → Good; 0.5–0.8 →
     Hard; <0.5 → Again). Само-оценка тут КАЛИБРОВАНА рубрикой → надёжнее свободной (мета-анализ: рубрика ↑).
  Это **structured-cloze/checklist self-assessment**, а не «доверься LLM». Опирается на generation (Chi) +
  немедленную сверку с source (RS-09 §3.4 — «почему»-карты безопасны только со сверкой). Уверенность: **[С]**
  (композиция из A-первичек; величина надёжности именно этой связки — не измерена, гейт §7).
- **Где возможно — конвертировать прозу в structured-cloze/key-point-reveal**, чтобы сигнал стал объективнее
  (RS-10 `verify: exact`). «Чистая» свободная проза остаётся, но её grade — из checklist, плюс отложенный
  человек-census на опасных темах (RS-10 Stage 5).

### 3.6 Метрики экспертного прогресса: генерация / transfer / калибровка (Q4)

Все — на клиентских логах (xAPI-подобные verb-события RS-10 §3.4; GDPR-минимизация RS-05 G7). Надстройка над
RS-01 §2.4 (log loss / RMSE(bins) калибровка планировщика) и RS-09 §6.

1. **Генерация (produce, не recognize).** Accuracy по генеративным типам (write-missing / find-the-bug /
   explain-checklist), **взвешенная к генеративным** (MCQ/cloze — вход, не мерило). Событие: `answered{cardType,
   correct, rating, hints, latency}`. Дополнительно — **support-level, на котором достигнут mastery** (см. §3.3):
   учащийся, освоивший атом на уровне `write`, экспертнее того, кто держится на `predict`. Это НОВАЯ
   экспертиза-метрика, которую даёт expertise-reversal-планировщик бесплатно.
2. **Transfer.** *near* — faded→полная задача (тот же концепт, новый вход); *far* — карты-сравнения «какой X где»
   (RS-09 §6, условие: subgoal-метки + prompt-на-принцип); *PFL/double-transfer* (Schwartz&Martin 2004,
   наследие RS-09) — изредка новый под-концепт, мерим СКОРОСТЬ выхода на mastery (готовность учиться).
   Событие: `transferProbe{kind: near|far|pfl, correct, timeToMastery}`.
3. **Калибровка (metacognition).** Уверенность ДО ответа → **Brier score** / надёжность (RS-01 §2.4, RS-09 §6).
   Экспертиза = высокая точность + точная само-оценка (анти-illusion-of-competence, Karpicke RS-01 §2.2).
   Для прозы checklist-self-grade (§3.5) даёт вторую калибровку: расхождение (само-оценка − #hit/N) — прямой
   сигнал метакогниции. Событие: `calibration{predictedRecall, actual, selfScore, checklistScore}`.

Mastery-порог — BKT-style **P(L)≥0.95** (RS-05 U6), считается на генеративных исходах, не на узнавании.
Анти-Goodhart (RS-05 G2): streak/DAU — отдельно, НЕ выдаются за понимание.

### 4.2 Автогрейд прозы (Q3) — что в волну 1
| Вариант | Надёжность (первичка) | Риск | Волна |
|---|---|---|---|
| rubric + LLM-judge (grade→FSRS) | QWK 0.585–0.64, **mid-range >2 ошибок**, over-score слабых (§3.4) | заучит ошибку (hypercorrection), authority bias | **НЕТ (только не-блок. сигнал)** |

## 6. Противоречия источников

- **LLM-judge «good agreement» (QWK 0.67–0.80) vs «близко к нулю»/mid-range >2 ошибок.** Не конфликт, а
  зависимость от задачи/уровня: на total-score крайних ответов — прилично; на criterion-level и СРЕДНИХ
  ответах — разваливается (§3.4). Консервативно берём худший случай (mid-range), т.к. именно он доминирует у
  растущего джуна и именно он кормит FSRS-интервал.
- **Self-assessment r≈0.59 (умеренно надёжно) vs «слабые завышают».** Оба верны — модератор: achievement level
  и наличие рубрики. Разрешение: **не свободный self-grade, а checklist с рубрикой** (§3.5), где именно
  рубрика поднимает надёжность.
- **FSRS «бинарен» (Again vs остальные) vs «4 уровня обязательны».** Оба верны: вход — 4 уровня, но модель
  памяти различает fail/pass бинарно + модуляцию S по Hard/Good/Easy. Для генеративных айтемов это удобно:
  объективный pass/fail даёт ось, процессные сигналы — модуляцию (§3.2).
- Манипуляций/инъекций в источниках (OSR-репо, Wiley/arXiv/Springer, EDM) не обнаружено; SEO/агрегаторы в
  источники не брались.

## 7. Что НЕ удалось выяснить (открытые вопросы / гейты)

1. **Learning-outcome expertise-reversal-планирования** — спайк доказал только МЕХАНИКУ связи S↔support и
   авто-реинфляцию; польза для обучения (лучше ли статичной колоды) — не измерена → **AB/sim-harness на людях**
   (RS-09 §11 Q4). Поведенческая модель исхода в спайке — грубая заглушка.
2. **Пороги S_MODIFY≈7 / S_WRITE≈21** — дизайн-ручки, не выведены из данных; калибровать метриками §3.6
   (доля лапсов после fade; time-to-mastery на уровне write).
3. **Надёжность именно связки «structured self-explanation + key-points checklist» как grade-сигнала** — не
   измерена прямым исследованием; r≈0.59 относится к само-оценке вообще. Нужен пилот: сравнить checklist-score
   с человек-грейдом на 30–50 прозаических ответах.
4. **Валидированного порога «доля LLM-грейда прозы без ревью»** — НЕТ (RS-05 §7.3 подтверждён числами §3.4).
   Нет «AI-исключения»: опасные темы — census-ревью человеком (RS-10 Stage 5).
5. **Что считать `slow` / soft time budget** для латентности-модулятора (§3.2) — зависит от типа карты и
   длины кода; калибровать на распределении латентностей когорты (RS-05 §3.2 latency-distribution).
6. **Easy из само-отчёта vs авто** — оставлен как явная кнопка; не искажает ли это S при честном «тривиально» —
   проверить на логах (риск инфляции интервалов).

## 8. Рекомендация (почему лучше альтернатив, со ссылками)

**Волна 1:**

1. **Grade генеративных айтемов = объективный исход, модулированный процессом (§3.2, спайк исполнен).**
   `correct=false→Again`; `correct=true & firstTry & !slow→Good`; struggled/hints/retry→Hard; `Easy` — только
   явный само-отчёт. Для code/predict/cloze `correct` берётся из детерминированного G-EXEC/exact (RS-10), НЕ из
   мнения LLM. Почему лучше: не геймится (RS-05 G3), не recognition (RS-01), кормит FSRS корректно (Again=fail,
   Hard=честный трудный pass — OSR-предупреждение соблюдено).

2. **Проза «объясни» = structured self-explanation + key-points checklist self-grade (§3.5).** Grade = `#hit/N`
   по пунктам из `depth.mechanics/spec` lesson-as-data, с немедленным reveal-сравнением. LLM-судья — только
   не-блокирующий флаг на человек-ревью, НЕ источник grade. Почему лучше rubric+LLM-judge: у LLM mid-range
   degradation (>2 категориальных ошибки на средних ответах против ~0.2 у человека, arXiv 2605.07647) + over-score
   слабых + authority bias (RS-05 U12) → заученная ошибка и hypercorrection (RS-05 §3.3); а рубрика поднимает
   надёжность само-оценки (мета 160 статей/29k участников, EPR 2023). Где возможно — конвертировать прозу в
   structured-cloze (`verify: exact`) для объективности.

3. **Expertise-reversal-планирование = S-пороги predict→modify→write (§3.3, спайк исполнен), пороги под AB.**
   Support-level выводится из единого FSRS-stability и авто-реинфлируется после лапса (спайк: S 32→2.40 → write
   откатывается к predict). Почему лучше статичной колоды: снимает леса по мастерству (Kalyuga expertise-reversal),
   что RS-09 назвал единственным путём к far-transfer и анти-упрощением; при этом НЕ вводит вторую модель
   состояния. Заявляем как ДИЗАЙН с эмпирическим гейтом (§7.1), не как доказанное «лучше».

4. **Метрики экспертизы = генерация (взвешенная к генеративным) + support-level-at-mastery + transfer
   (near/far/PFL) + калибровка (Brier + checklist-расхождение) (§3.6).** Mastery = P(L)≥0.95 на генеративных
   исходах. Streak/DAU — отдельно (RS-05 G2). Почему лучше «accuracy+streak»: меряет produce/transfer/калибровку —
   экспертизу, а не иллюзию (Karpicke, Schwartz&Martin — наследие RS-09 §6).

**Что спайкнуть дальше (эмпирические гейты, НЕ волна-1-константы):**
- AB/sim-harness пользы expertise-reversal-планирования и калибровки порогов S (§7.1–7.2);
- пилот надёжности checklist-self-grade против человек-грейда на 30–50 ответах (§7.3);
- калибровка `slow`-порога на латентностях когорты (§7.5).

Всё держится на рецензируемой первичке (Kalyuga, Chi, Andrade-мета, ASAG-2024–26) + исполнении реальной
ts-fsrs, а не на маркетинге. Единственные узлы средней уверенности (надёжность checklist-связки, пороги S,
learning-outcome планировщика) явно вынесены под эмпирические гейты — анти-slop-дисциплина сохранена.
