# Ревью · Линза «Вовлечение и ежедневная петля»

Продукт: Telegram Mini App — ежедневный тренажёр для сеньоров (трек C#).
Оценщик: внешний скептик. Раунд: 1. Дата: 2026-07-11.

## ВЕРДИКТ — ПРИНЯТО С ОГОВОРКАМИ (петля работает, но «минута смерти» наступает быстро из-за отсутствия chaining сессии)

Сборка зелёная: `npx tsc --noEmit` → exit 0; `npx vite build` → built in 217ms, 0 ошибок.
State machine дома чистая, числа честные (сверил XP/streak с бэкендом), дизайн-язык применён верно.
Но как ежедневная петля продукт **недожат в одном ключевом месте**: обещание «сессия из 7 карточек» не подкреплено механикой непрерывной сессии.

**Паспорт вкуса: PASS** — крем #F6F1E9 + коралл #F0533A + Rubik + спарк-знак присутствуют и применены дисциплинированно; нет золота/градиентного слопа/streak-shaming. Доказательство: `docs/evidence/product-readiness/390-first-run.png`, `390-done.png`, `390-session.png`. Signature-элемент (спарк-знак в топбаре + коралловый акцент) заметен на каждом экране.

## ГЛАВНОЕ (P0/P1)

### P0-1 · «Сессия» не является сессией — нет chaining, каждая карточка = round-trip домой
Локация: `app/src/app/lessonRunner.ts:522` (`lesson.cards[0]` — рендерится ровно ОДНА карточка) + `:814-822` (`appendNext` даёт единственную кнопку «На главную») + `app/src/app/router.ts:63` (`showLesson` открывает один урок).

Каждый урок содержит ровно 1 карточку (проверено: boxing/value-vs-reference/gc → 1 `question:` каждый). Значит due-очередь «7 карточек» = до 7 РАЗНЫХ уроков. Но после оценки карточки единственный путь дальше — «На главную», затем пользователь должен ВРУЧНУЮ снова нажать герой. Нет «Следующая карточка», нет автопродвижения по очереди, нет счётчика «2 из 7».

Эффект на петлю: hero обещает «7 карточек · ~7 минут» и «Начать сессию», но механика выдаёт их по одной с полным возвратом на главную между каждой. Это ломает flow-state и главный крючок удержания «ещё один разбор». Сеньор, у которого 5 минут, после первой карточки видит домашний экран как «стоп-сигнал», а не «продолжай». Заявка «сессия» частично display-only: кнопка есть, непрерывной сессии за ней нет.

Фикс: после `#gradeDone` показывать «Следующая карточка (2/7) →», ведущую в следующий due-урок без возврата домой; счётчик прогресса сессии в топбаре; «На главную» — вторичной ссылкой. Домой возвращать только когда очередь пуста → тогда показать done-hero. Это превращает 7 отдельных заходов в одну сессию.

### P1-1 · Done-hero — тупик без действия, когда завтра пусто
Локация: `app/src/app/home.ts:371-384` (`doneHero` не содержит `<button class="cta">`), строки `strings.ts:68` (`doneTomorrow(0)` = «Завтра новых повторов пока нет — можно взять свежий урок»).

В состоянии «День закрыт» когда `tomorrowDue === 0`, копирайт прямо говорит «можно взять свежий урок», но КНОПКИ взять этот урок нет — только чипы XP/серия + «возвращайся завтра». Пользователь прочитал приглашение к действию и не может его выполнить, не поскроллив вниз к списку тем. Для линзы удержания это упущенный «ещё один разбор» ровно в момент максимальной мотивации (только что закрыл день, дофамин на пике).

Фикс: в `doneHero`, когда есть непройденные уроки (`nextUnseen`), добавить вторичную CTA «Взять свежий урок →» под `done-tomorrow`. Не давить (день уже закрыт — это опционально), но дать affordance под уже написанное обещание.

### P1-2 · Empty-new и empty-all визуально почти неотличимы
Локация: `app/src/app/home.ts:388-404` (`emptyHero`) + скриншоты `390-empty-new.png` vs `390-empty-all.png`.

Оба экрана: одинаковая структура (kicker/title/lead/CTA), одинаковый герой-стиль, разница только в тексте и в оттенке kicker (коралл vs sage). «Фундамент пройден целиком» — это КУЛЬМИНАЦИЯ трека, момент достижения, а выглядит как рядовое «нет карт к повтору». Достижение не отмечено визуально (нет печати/seal как в done-hero, нет акцента завершения).

Фикс: для `empty-all` добавить `done-seal`-подобный элемент завершения трека и более праздничный тон (это редкое событие — заслуживает веса). Сейчас важнейший milestone продукта проходит незаметно.

## ЧТО ПЛОХО (обязательная секция, ≥3)

1. **Reward-момент после оценки тихий.** `lessonRunner.ts:788` — после грейда показывается только текст «Прогресс сохранён… следующий повтор через N». Прогресс-бар добегает до 100% (`bumpTo(100)`), есть `tg.notify("success")` (хаптик), но визуального «+10 XP» всплеска в момент грейда нет — XP-чип бампится (`xpBump 0.5s`) только на просмотре сегментов (+2 каждый), а сам грейд (главное событие) XP-фидбека в UI не даёт. Кульминация урока недо-вознаграждена. (P2)

2. **Серия «1 день» на всех состояниях — потолок не показан.** Во всех done/empty скриншотах «Серия · 1 день». Логика honest (`Db.cs:376` — consecutive UTC days), но UI нигде не показывает «завтра будет 2» как forward-hook для серии. Milestone-celebration только на [3,7,14,30,60,100] — между ними серия молчит. Для линзы удержания серия — второй по силе крючок после due-очереди, и он почти немой в диапазоне 1-2 дня, где отвал максимален. (P1-адъяцент, вынес в «что плохо»)

3. **Онбординг-hero одноразовый, но нет «повторного тура».** `onboarding.ts` + `home.ts:222` — `markOnboarded()` навсегда прячет интро при первом старте урока. Механизм FSRS объяснён в интро и в профиле (`howItWorksBody`), но новый пользователь, пропустивший интро («Осмотреться самому»), теряет объяснение ценности навсегда с главной. Мета-прозрачность спрятана в Профиль. (P2)

4. **«~7 минут» может обмануть ожидание.** `sessionMeta` показывает суммарную оценку, но с учётом round-trip-механики (P0-1) реальное время выше из-за навигации между уроками. Оценка честна для контента, но не для фактического UX-пути. (P2, снимется фиксом P0-1)

## ПРОВЕРКА ДОКАЗАТЕЛЬСТВ (заявкам builder-а не верю — перепроверил)

- **«Числа из живого бэкенда, ничего не выдумано»** — ПОДТВЕРЖДЕНО частично измерением. XP: `home.ts:161` `xpToday = todayCount*10` совпадает с сервером `Program.cs:219` `xp = reviewsTotal*10`. «+70 XP за сегодня» на done-скрине = 7 повторов × 10 — честно. Streak: `Db.cs:356-391` — реальный consecutive-days счёт. Лжи в числах не нашёл.
- **«Streak supportive, never shaming»** — ПОДТВЕРЖДЕНО. Нет красного, нет «вы потеряли серию», при 0 — «Начни серию сегодня» (`strings.ts:84`). RS-01 соблюдён.
- **«5 состояний home как state machine»** — ПОДТВЕРЖДЕНО чтением `deriveHomeState` (`home.ts:75-81`), pure-функция, порядок корректен.
- **empty-new/empty-all скриншоты** — сделаны через `__forceHomeHero` (форс-рендер реального героя, `home.ts:251`), т.к. no-activity день не создать за одну сессию против живого бэка. Это РЕАЛЬНАЯ разметка, не мок — принимаю, но это не полный e2e для этих двух состояний.

## СОМНЕНИЯ (не смог измерить сам)
- Бэкенд .NET + живая БД не запускались мной (нет dotnet-рантайма в песочнице) — done-hero серию кадров «≥5 за 500мс после события» глазами на живом продукте не снял; сужу по коду + статик-скриншотам loop.mjs. Дифференциацию фидбека разных грейдов (Good vs Again анимация) вживую не проверил — по коду `revealGrade` (`:732`) отличается баннером ok/no, но серию кадров не снял.
- Реальную «минуту смерти новизны» против таймлайна ТЗ не мерил sim-harness-ом вживую (harness `loop.mjs` есть, но требует живой бэк). Вывод о round-trip основан на статическом анализе пути кода — он однозначен, но телеметрию флоу не снял.

## ЧТО ХОРОШО (кратко)
- Прогресс-экран (`390-progress.png`) — сильнейшая мета-прозрачность: калибровка, тепловая карта 4 недели, прогноз FSRS на 7 дней, разбивка оценок. Это именно то, что удерживает вдумчивого сеньора. Отличная работа.
- Честное разделение «прохождение материала» vs «закрепление карточек» (`strings.ts:32-35, 122-137`) — не путает viewing с mastery, редкая интеллектуальная честность.
- Калибровочный фидбек (`calibrationMessage`, `:514`) — умный крючок для сеньора: «переоценил» / «доверяй себе больше» цепляет метакогнитивно.
- Skeleton-загрузка (`390-loading-skeleton.png`) в форме реального фрейма — ничего не прыгает. Error-card с деталью и рабочим retry — аккуратно.
