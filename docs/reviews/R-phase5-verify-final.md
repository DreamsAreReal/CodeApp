## Фикс-волна P2 (закрыта в рамках VERIFY — чтобы вердикт был без единого `partial`)
1. **rate-limit без исполняемого теста** (единственный `partial`-гейт) → добавлен детерминированный тест 429.
2. **дубль подписи калибровки** в `progress.ts` (нашёл consumer) → убран.
3. **viz-fit ассертил `≥1`** вместо `6/6` → порог ужесточён (регресс миграции теперь ловится).

## Backlog P2 (НЕ блокеры — вынесено в следующую волну, с ценой)
- Точность: закавыченные КОМПОЗИТЫ через «…» помечать явно; заморозить цитаты по git-permalink/архиву (живые MS-доки дрейфуют); CLRS — book-источник непроверяем внешним fetch.
- Бэкенд: golden-вектор — фиксированный снимок (не property-based против живого py-fsrs); нет multi-user concurrency и review+delete-гонки; конкурентный тест ловит слово `locked` (слабый ассерт).
- CI: viz-fit гоняется против dev `dotnet run`, а не prod-образа; coverage-порог не enforced; live-loop харнесы (run.mjs) вынесены TODO; deploy не ждёт `healthy` перед prune.
- A11y: `axe .include('#app')` не покрывает document-level (html-lang/title).
- Контент волны 1: 6 из ~10–12 уроков C#-ядра (пользователь поставил на паузу).

## Ретро (что сработало / уроки)
- **Внешний чек > self-pass**: оценщики перепроверили ИСПОЛНЕНИЕМ (независимый re-run py-fsrs, docker healthcheck, 11 fetched источников), а не приняли зелёное на веру — риск №1 (точность) закрыт первично.
- **Профилактика на входе строителя** (автолейаут `at`, viz-fit AUTHORING-PROOF) сделала «кривой кадр» невозможным по конструкции — не ловим ревью, а исключаем на входе.
- **Развод по ресурсам в фан-ауте** (бэкенд-тесты→фронт-харнесы последовательно, docker/accuracy параллельно, consumer после освобождения портов) избежал гонок портов/сборки.
- **Урок**: consumer со свежим взглядом нашёл дубль-лейбл, который 5 волн self-pass пропускали — незамыленный проход окупается.
