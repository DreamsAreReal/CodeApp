# RS-00 — Брейншторм (lite: 5 углов, серийно)

North Star: пользователь открывает бота в Telegram → Mini App открывается и работает всегда.

## Углы

1. **Пользователь бота («турист»)**. Жмёт кнопку меню — либо приложение, либо белый
   экран без объяснений (TG не показывает TLS/DNS-ошибки внятно). → Следствие для ТЗ:
   серт с автопродлением; проверка снаружи (curl с чужой сети) — гейт, а не «у меня открылось».
2. **Сосед по серверу (владелец xray/portfolio/zhaba)**. VPS общий; xray уже держит
   8443/2096, кто-то может рассчитывать на 443. → Следствие: перед занятием 443 доказать,
   что он свободен (ss + firewall + конфиги соседей); правки nginx — отдельным
   server-блоком, бэкап + `nginx -t` перед reload; чужие конфиги не трогать.
3. **Скептик («зачем домен, есть же туннели»)**. cloudflared/tailscale funnel дали бы
   HTTPS без DNS-возни. → Контраргумент: ещё один постоянный демон и зависимость от
   третьей стороны; TG-кнопка требует стабильный URL. Рассмотреть в design как fallback,
   если с доменом затык.
4. **Пессимист («где провалится»)**. (а) 443 может быть закрыт на уровне облачного
   firewall провайдера — проверить снаружи, не только ss; (б) домена может не быть,
   а DuckDNS требует аккаунт пользователя → вопрос на гейт; (в) редеплой через deploy-hub
   может пересоздать контейнер с другим портом → проверить, зафиксирован ли 8091 в
   конфиге хаба на VPS; (г) self-signed/IP для TG Mini App не работают — только валидный
   CA-серт; (д) setChatMenuButton перезапишет текущую кнопку бота — проверить текущее
   состояние перед записью; (е) серт выпущен, а продление не настроено → через 90 дней
   тихая смерть (см. угол 1).
5. **Владелец продукта (планка «вау»)**. Ступень 3 здесь = не «страничка открылась», а
   надёжный вход: автопродление серта, порт прокси стабилен при редеплоях, HTTP→HTTPS
   редирект, security-заголовки не ломают встраивание в TG (X-Frame-Options/CSP — TG
   рендерит в WebView, заголовки anti-frame могут убить Mini App!). Бонус: тот же домен
   даёт обычный веб-доступ для шаринга.

## Следствия для ТЗ (синтез)
- Гейты формулировать как внешние замеры: HTTPS снаружи, валидность серта, работающий
  Mini App URL у бота, автопродление доказано (dry-run), соседние сервисы живы до/после.
- Открытый вопрос пользователю (гейт): домен — свой / DuckDNS (нужен его аккаунт) / другое.
- Риск-чек до сборки: 443 свободен изнутри И снаружи; порт 8091 стабилен в deploy-hub.
- Анти-frame-заголовки в nginx НЕ ставить (сломают WebView Telegram).
