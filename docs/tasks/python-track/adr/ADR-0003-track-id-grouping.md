# ADR-0003 — track id «PY» + группировка home через TRACK_GROUPS

## Статус
Принято 2026-07-15

## Контекст
Треки на бэке — свободные строковые ID из seed (LessonStore.cs:40 парсит `track`);
C#-уроки живут в T1/T2 и рендерятся одной лентой с хардкод-лейблом «Путь · Ядро C#»
(strings.ts:25; home.ts:132 без фильтра). Требование гейта: разделы C# и Python
раздельны и не пересекаются.

## Рассмотренные варианты
| Вариант | Плюсы | Минусы | Замеры из спайка |
|---|---|---|---|
| A. track=`PY` + фронт-реестр TRACK_GROUPS {csharp:[T1,T2], python:[PY]} | читаемый id; T1/T2 не переименовываются (сид/прогресс целы); группировка = чистая функция от реестра | id вне T-нумерации куррикулума | Explore: бэк принимает произвольный track (LessonStore.cs:40, `/api/lessons` вернёт его); прогресс per-lesson, от формата id не зависит |
| B. track=`T8` (продолжить нумерацию) | единая схема | нумерация RS-04-куррикулума (T1–T7) не включает Python — T8 столь же произволен, но менее читаем | те же |
| C. Переименовать C# в общий `CS` и вводить `PY` | симметрия | миграция id уроков = миграция review_state/прогресса на проде — риск данных без выгоды | нет (отклонён без спайка: трогает прод-данные) |

## Решение
Вариант A: `PY.M<n>.<slug>`; фронт группирует секции по TRACK_GROUPS (лейблы из
strings.ts, хардкоды убираются). Порядок секций: C# (существующий) → Python; внутри
секции порядок LESSONS.

## Последствия
- Добавление трека N+1 = строка в TRACK_GROUPS + data-файлы (масштабируется).
- Принятые риски: смешанная схема id (T1/T2/PY) — косметика, контракты не задевает.
