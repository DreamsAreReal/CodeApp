# ADR-0002 — G9-пейсинг: sim-harness полосы успеха = N/A для контентной волны

## Статус
Принято 2026-07-15

## Контекст
Продуктовый гейт G6 «полоса успеха сессии 83–88%» предполагает sim-harness. Критик
(R-brief) потребовал вернуть гейт или явный N/A с обоснованием.

## Рассмотренные варианты
| Вариант | Плюсы | Минусы | Замеры из спайка |
|---|---|---|---|
| A. N/A + lead-прокси + lag-метрика | честно; прокси измеримы сейчас | полоса успеха узнаётся только на живых повторах | grep app/verify: sim-harness полосы успеха в репо ОТСУТСТВУЕТ (проверено 2026-07-15; список харнессов в GUIDE §5 подтверждает) |
| B. Построить sim-harness знаний | прямое измерение | симуляция «знаний сеньора» = выдумка модели, не замер; новый инструмент против скоупа волны | нет |

## Решение
Вариант A. Lead-прокси в воротах (лесенка predict→modify→explain у 100% уроков;
estMinutes по калибровке boxing 7 сегм ≈ 9 мин; флагман ≤12 мин). Lag: полоса успеха
83–88% по correct/confidence из progress_events после релиза (метрики успеха ТЗ).

## Последствия
- Пейсинг-риск контролируется прокси и чекпойнтом 1-го урока (глазами).
- Принятые риски: реальная сложность карточек может выйти из полосы — увидим по lag-метрике,
  правится следующей волной (перекалибровка карточек, не движка).
