# ADR-0001 — считать эталоны Python-карточек локальным python3.12 при авторинге

## Статус
Принято 2026-07-15

## Контекст
Карточки predict-output требуют `verify.expect` = реальный stdout (ТЗ G2/G2a). Для C#
эталон считает авторский dev-эндпойнт `/api/authoring/run-csharp` (Roslyn). Python-раннера
на бэке нет. RS-01 изначально предлагал endpoint `/api/run-python`; критик потребовал ADR.

## Рассмотренные варианты
| Вариант | Плюсы | Минусы | Замеры из спайка |
|---|---|---|---|
| A. Авторский локальный python3.12 (файлом) + лог в evidence | 0 кода на бэке; версия пиннится; census-лог воспроизводим | нет self-serve для будущих авторов без python3.12 | RS-03: 29 спайков прогнаны python3.12.13, лог evidence/spikes/run-log-py312.txt; c25/c26 доказали версионный риск 3.9↔3.12 |
| B. Endpoint /api/run-python | симметрия с run-csharp | код+песочница+безопасность на бэке; снятие LOCKED без нужды; в проде всё равно 403 | нет (спайк не делался — вариант отклонён по цене/риску) |
| C. MCQ-only (без exec) | ноль работы | теряется генерация (ядро методики RS-09), Гудхарт | RS-01 D-3: MCQ-fallback 1/27 ≈ 4% — exec почти всюду возможен |

## Решение
Вариант A. Эталон = stdout `python3.12 <card>.py` (файлом, не -c/REPL — интернирование
констант различается, доказано int_cache_file.py: True/True/False/True), ×2 прогона,
stdout/stderr раздельно. `verify.run` в карточке — описательная строка `python3.12 card.py`.

## Последствия
- Проще и безопаснее; авторинг будущих волн требует локального python3.12 (зафиксировано
  в AUTHORING-доке волны).
- Принятые риски: минорные версии 3.12.x могут менять тексты сообщений — цитаты сообщений
  об ошибках печатаем через `type(e).__name__`, не traceback-текст (чеклист RS-03).
