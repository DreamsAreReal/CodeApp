## ПРОБЛЕМЫ

- **[СУЩЕСТВЕННОЕ · ПРЕ-СУЩЕСТВУЮЩЕЕ] POST /api/review принимает НЕсуществующий itemId →
  HTTP 200 и ПЕРСИСТИТ фантомную карту.** `{"itemId":"BOGUS.M99.nope/c1","grade":3}` →
  200, создаётся review_state + progress_event. 6 запросов на выдуманные items
  (`FAKE.1/cX`…) → `/api/stats` reviewsTotal 0→6, **XP 0→60** (`api-edges-log.txt`).
  Нарушение целостности: сервер не валидирует itemId против каталога (`Program.cs`
  MapPost /api/review + `ReviewService.Review` — нет `store`-lookup). Смягчение: auth
  партиционирует по tgId → пользователь раздувает ТОЛЬКО свою статистику (не кросс-юзер);
  реальный клиент фиктивных id не шлёт; фантомные items НЕ попадают в `/api/due`
  (каталог-гейт, due остаётся 59). Это product-код, НЕ добавлен волной Python → не
  регрессия волны и не блокер деплоя этой волны. Фикс: 404/400 на itemId вне
  `LessonStore`. → в backlog продукта.

## ЧТО ПЛОХО

- **Быстрая полная сессия ловит honest 429 → консольная ошибка.** Phase B (60 карт за
  267с ≈ 4.5с/карту) дала 3× HTTP 429 (2 review + 1 lesson-progress) — лимитер 60
  мутаций/мин (by-design, RejectionStatusCode 429). Сессия НЕ теряет данные: UI
  показывает «save failed» + ретрай работает (после 62с окно сброшено, review прошёл;
  reviewErrors итог=0, due дренировался в 0). Но всплывает сырым `console: Failed to
  load resource: 429`. На человеческом темпе (15–40с/карту на predict-output) 60/мин
  недостижим → реального вреда нет; это артефакт робота. Амплификатор — day-one сид
  выкатывает ВСЕ 59 карт как due сразу (нет дозирования new/день): брифовый риск
  «due-очередь дня после сида не взрывается» фактически не выдержан (59 в первый день).
  lesson-progress ест бюджет review-лимита — стоит выделить отдельный лимитер или не
  лимитировать прогресс-репорты.
- **500 вместо 400 шире, чем документировал E1.** E1 назвал только grade="Good".
  Перепроверил границу: 500 дают также — пустое тело, grade нечисловой строкой,
  itemId=null (grade валиден, падение в DB), битый JSON. Числовой grade вне диапазона
  (99/-1/0) и пустой `{}` → корректный 400. Гвард `req.Grade is <1 or >4` срабатывает
  только после успешного биндинга; ошибки десериализации летят в ExceptionHandler
  (problem+json, сервер не падает). Лог-шум прода. (`api-edges-log.txt`)
- **Двойной review одного item на API не идемпотентен** (reps 1→2, Learning→Review,
  due +2д) — by-design FSRS; защита от двойного сабмита живёт на клиенте (доказано:
  grade-double-tap = ровно 1 POST). Но контракт эндпойнта голый: два быстрых легитимных
  POST дважды двигают расписание.

## ЧТО ВЫДЕРЖАЛО (доказательства — перепрогнаны, не на веру)

1. **Ввод карточек (`input-fuzz-log.txt`, ALL GREEN):** пустой → submit disabled;
   пробел/5000-симв/юникод-эмодзи-RTL/ANSI/wrong-case → корректно «мимо»;
   CRLF+хвостовые пробелы и ведущие/хвостовые пустые строки → нормализованы в «верно»;
   5000 символов обработаны за 546мс; **SQL/HTML-инъекция экранированы** (`__pwned`=null,
   разметка выведена текстом `&lt;img…&gt;` — `fuzz-html-injection.png`). 0 конс. ошибок.
2. **Гонки UI (`ui-races-log.txt`, ALL GREEN):** тройной клик submit = 1 вердикт + lock;
   grade double-tap = **ровно 1 POST /api/review**, due 59→58; Next-спам в FLIP и
   скраббер-трэш в автоплее — индекс в границах; переключение трека сразу после входа в
   урок — home побеждает, нет ghost-DOM (nav-токен-гвард); abandon посреди карты
   (отвечено, не оценено) — due не двинулся (нет фантомного review); reload посреди
   predict-гейта — чистый бут, урок переоткрывается.
3. **Сессия (`session-log.txt`):** interrupt+re-enter ×3 — счётчик «N из M» = живому
   server-due каждый раз (59→57→55→53); Again-реэнкью 59→60, **cap ×1 держится** (второй
   Again не растит очередь); счётчик совпал на всех 60 картах; heap 10→10 МБ (нет течи);
   очередь дренировалась в 0.
4. **Вьюпорты (`viewports-rm-log.txt`, ALL GREEN):** 320×568, 430×932, landscape 844×390
   × {M8, M11} — 0 горизонтального overflow, 0 клипа SVG-узла за viewBox, 0 конс. ошибок.
5. **Reduced-motion (ALL GREEN):** все сегменты на статичном финальном кадре без
   взаимодействия; многострочная консоль рендерит 3 строки (фикс b3c50a4 виден, не
   клипается); ghost opacity 0.4 присутствует; 0 конс. ошибок. Подтверждено глазами.

## ПРОВЕРКА ДОКАЗАТЕЛЬСТВ

- Известный «500 вместо 400» (E1) → воспроизведён и уточнён: 500 = ошибки биндинга
  (тело/тип/null/битый JSON), НЕ числовой grade вне диапазона (тот 400). Граница шире
  заявленной E1.
- «Холодный offline» (E1) → вне моего чеклиста (проверял тёплый путь через уроки), не
  дублирую.
- b3c50a4 «многострочный stdout построчно» → перепроверено: 3-строчная консоль в M8/M11
  reduced-motion не клипается (`rm-*-final.png`).

## СОМНЕНИЯ

- Day-one 59-карт-очередь: брифовый пункт про дозирование new/день фактически не
  выполнен (все new сразу due). E1 тоже видел 59 и принял. Это влияет на длину сессии и
  близость к rate-лимиту — решение продукта, не блокер волны.
- 429 в Phase B — граница «честный лимит vs UX-заусенец»: классифицировал МЕЛОЧЬ, т.к.
  на человеческом темпе недостижимо и деградация graceful (ретрай, без потери данных).
  Если продукт всерьёз продаёт «одна сессия на весь день» при большой очереди — стоит
  поднять до СУЩЕСТВЕННОЕ на ретро.
