## 2. ЧТО ПЛОХО → MUST-FIX (каждый проверяем)

1. **brief.md устарел против решений гейта — рассинхрон design↔brief.** Гейт исключил
   мостики C#↔Python; design/features везде «БЕЗ сравнений с C#» (общая приёмка (е)),
   а brief ДО СИХ ПОР: «Ставка вау: … + мостики C#↔Python (у каждого Python-урока —
   кадр/врезка "как это в C#")», «Signature-элемент: dis + мостики», P0-строка
   «py-closures-scope … — зеркало C#-closures». Evaluator, мерящий против брифа,
   завалит уроки за ОТСУТСТВИЕ мостиков, builder — за их наличие. Fix: обновить
   Ставку вау/Паспорт/P0-строки брифа (ось остаётся «сеньор видит невидимое» —
   она и без мостиков единственная и конкретная) + отразить решения гейта (11+2,
   собес-флоу сохранён). Проверка: grep «мостик|как это в C#|зеркало C#» по brief = 0
   вне раздела «Не входит».
2. **F14 — критерии-действия, не исходы**: «скрин отревьюен», «решение задокументировано»
   выполняются фактом потраченного времени. Fix: «[ ] Прогресс с PY-уроками не смешивает
   треки (скрин: секции/подписи различимы) ЛИБО оформлен cut по протоколу с причиной»
   — бинарный исход. Проверка: в приёмке F14 нет глаголов «отревьюен/задокументировано»
   без наблюдаемого состояния.
3. **Шпаргалки vs общая приёмка (и) и ADR-0002.** Пункт (и) требует «лесенку
   predict→modify→explain», ADR-0002 — «лесенка у 100% уроков» (это lead-прокси G9!);
   F12/F13 дают только predict-колоду 4–6 карточек. Противоречие трёх документов.
   Fix: либо явно исключить M12–M13 из пункта (и) и из знаменателя ADR-0002
   («100% уроков-механизмов»), либо добавить в колоды modify/explain-ступень.
   Заодно: F1 заявляет «2 карточки», а пересчёт D-3 для names-objects — 3 (c01/c02/c03)
   — указать 3 или обосновать срез. Проверка: числа карточек в фичах == D-3;
   формулировка (и) не противоречит F12/F13.
4. **Две строки брифа остались без носителя в фичах.** (а) Проверка FSRS-дозирования
   («due-очередь дня после сида не взрывается — замерить и записать», брифовский риск,
   фаза VERIFY) — нет ни в одной фиче: прикрепить критерием к F15 или явным пунктом
   финального VERIFY (замер числа new/день в evidence). (б) P2-пункт брифа
   «GIL/free-threading 3.13 отдельным уроком» молча исчез (GIL-сегмент в F11 его не
   заменяет — это разные обещания): строка в Backlog STATE, не тихая потеря.
   Проверка: обе строки имеют адрес (фича/VERIFY-чеклист/Backlog).
