# RS-00 — Брейншторм: Python-трек из 00-python-refresher.md

Углы генерировались последовательно, каждый обязан отличаться от предыдущих.
North Star: сеньор-разработчик каждый день заходит в Mini App и ГЛУБОКО, с анимациями,
рефрешит фундамент — теперь Python (файл пользователя = блок 0 подготовки AQA).

## Углы → инсайты → следствия для ТЗ

1. **Сеньор в поездке (существующий кейс продукта).** Уроки самодостаточны, 5–10 мин,
   работают в готовой daily-петле FSRS. → Python-уроки = те же LessonData, никакой новой механики.
2. **AQA-собеседуемый (аудитория md).** Файл структурирован под собес («на собесе спросят /
   стоит отвечать так»). Это ЦЕННОСТЬ, не мусор: собес-вопросы = готовые misconceptions и карточки.
   → сохранить собес-фрейминг в карточках/misconceptions (решение тона — на гейте).
3. **Скептик «зачем».** Python-синтаксис есть везде; ценность трека только в (а) под-капотом,
   (б) анимациях каждого нюанса, (в) FSRS-петле. → справочные разделы (методы строк/коллекций)
   НЕ уроки — карточки или скип.
4. **Ментор-автор.** 15 разделов ≫ одна волна. md сам маркирует 6 «глубоких» механизмов
   (декораторы, генераторы, context managers, ООП, type hints, async) — это ядро нарезки.
   → волна 1 = 6 механизмов + фундамент модели памяти (имена→объекты, mutable/immutable).
5. **Инженер движка.** Примитивы переиспользуются: slot/ref/obj = «имена→объекты» (Python
   ВСЁ — ссылки на объекты кучи!), buckets/chain = dict/set, timeline = async/генераторы,
   gate = try/except и isinstance-проверки, IL-панель → панель `dis` (байткод CPython).
   → проверить кодом: подсветка hlcode.ts (заточена под C#?) и панель il — обобщаются ли.
6. **Методист (RS-09 продукта).** predict-output идеален для Python: эталон = реальный stdout
   `python3` (не run-csharp!). Лесенка predict→modify→explain уже поддержана. → авторский гейт
   G-EXEC-PY: каждый expect прогнан локальным python3, версия зафиксирована.
7. **Аудитор точности (риск №1).** md писан «для собеса» — есть упрощения; всё сверять с
   docs.python.org (reference/datamodel, library), PEP-ы; цитаты дословно EN. → банк
   первоисточников на каждый урок.
8. **C#-сеньор, изучающий Python (сам пользователь!).** В приложении УЖЕ есть C#-уроки тех же
   тем (value-vs-ref, GC, closures, async, hashtable). Мостики «в C# так — в Python эдак»
   (всё ссылки; нет value-типов; GIL vs потоки; async однопоточный event loop vs TPL) —
   уникальный крючок нашего продукта. → в уроках сравнения C#↔Python где естественно.
9. **Худшие условия.** Телефон, viewBox ~340px: длинные листинги md резать до 1–4 строк на
   сегмент; движок бросит overflow — это защитит.
10. **Продукт/инвестор.** Python-трек = первое доказательство мультитрек-тезиса (GUIDE §1).
    → track id и группировка home должны масштабироваться (проверить, как home делит треки).
11. **A11y.** Харнессы axe уже гоняются — новых требований нет, текст сегментов = обычный HTML/SVG.
12. **Конкурент (Python Tutor / Real Python).** Python Tutor визуализирует произвольный код,
    но без курации/запоминания. Наше отличие: курируемые нюансы + анимация КАЖДОГО + FSRS.
    → в уроках не «трассировка вообще», а прицельные gotcha-кадры.

## Свод следствий для ТЗ
- Волна 1: модель памяти Python (имена/объекты/mutable) + 6 механизмов md; справочник — не уроки.
- Каждое утверждение — первоисточник docs.python.org/PEP, цитаты дословно; expect = python3.
- Мостики C#↔Python как фирменная ось; собес-фрейминг — спросить на гейте.
- Техвопросы в DESIGN: track id, home-группировка, hlcode/il-панель для Python, seed.
