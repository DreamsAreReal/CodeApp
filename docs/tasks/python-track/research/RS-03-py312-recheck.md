# RS-03 — Перепрогон спайков на python3.12 (закрытие must-fix №1, №3 критика)

Раннер волны: **python3.12 (3.12.13, локально)**. Все 26 спайков RS-01 перегнаны
(`for f in c*.py; do python3.12 $f; done`, scratchpad/pyspike/). Дифф против 3.9.6:

## Разошлись только 2 из 26
- **c25 (`int | None`, PEP 604)**: 3.9 — TypeError при def (аннотации вычисляются);
  3.12 — работает, печатает `0`. Следствие: сегмент про PEP 604 подаём как «современный
  синтаксис (3.10+)», НЕ как «падает в рантайме». Экран ошибки 3.9 — только как
  историческая справка, без exec-карточки на этом.
- **c26 (dis)**: байткод 3.12 = `RESUME` / `MAKE_CELL` / `BINARY_OP` (не BINARY_MULTIPLY
  из 3.9). Все опкоды в уроках — ТОЛЬКО из реального `python3.12 -m dis`-прогона.

Остальные 24 спайка байт-в-байт совпали (в т.ч. mutable default `['a'] ['a','b']`,
late binding `[2,2,2]→[0,1,2]`, finally-return, MRO `B`, mangling AttributeError,
contextmanager setup/42/teardown, for-else, gather speedup >3x).

## Малые int (must-fix №3) — исполнено файлом на 3.12
`int_cache_file.py`: `257 is 257` в ОДНОМ файле → **True** (интернирование констант
co_consts при компиляции); `int("257") is int("257")` → **False** (рантайм-объекты,
кэш только −5..256); `int("256") is int("256")` → **True**. Классический «257 → False»
верен лишь для рантайм-созданных значений; поведение частично implementation-defined.
Правило: сегмент подаётся с конструкцией `int("257")`, exec-карточка на identity малых
int ЗАПРЕЩЕНА (недетерминированно между способами исполнения).

## Прочие must-check
- Текст UnboundLocalError изменён в 3.11+ — все цитаты сообщений об ошибках в уроках
  берём из реального вывода 3.12.
- c19: RuntimeWarning «coroutine was never awaited» уходит в **stderr** — карточки
  сверяют только stdout; предупреждение в уроке показывать как отдельный кадр, не в expect.
- Замер «40.3 MB» из RS-01 (генератор vs список) — перемерить на 3.12 при авторинге
  сегмента, число в урок только из прогона.

## Чеклист допустимого `verify.expect` (G-EXEC-PY, анти-Гудхарт, must-fix №5)
Expect обязан быть: (1) детерминирован между прогонами и способами запуска (файл);
(2) только stdout (stderr отрезается); (3) без печати set/dict-порядка, где он не
гарантирован (set — запрещён в expect); (4) без таймингов/адресов/id()/путей;
(5) без `is` на рантайм-int/str вне гарантий; (6) без текста traceback (только имя
исключения, печатаемое через `type(e).__name__`); (7) прогнан python3.12, версия и
лог — в docs/tasks/python-track/evidence/. MCQ-fallback (карточки без expect) — ≤25%
карточек волны, только для compare/explain.
