# ТЗ — Python-трек: волна 1 (уроки из 00-python-refresher.md)

## Цель
В деплоенном приложении появляется трек «Python»: N глубоких анимированных уроков
(по нарезке RS-01) на планке продукта (плотность boxing-эталона, 100% утверждений с
первоисточником, карточки с реальным stdout python3), home показывает два трека,
все харнессы зелёные, задеплоено на VPS через CI.

## Контекст
Продукт готов (ступень 3, verified, задеплоен): движок living-diagram + lesson-as-data +
FSRS-6; контент — 6 C#-уроков. Пользователь положил конспект `00-python-refresher.md`
(«Блок 0/11 — Python-фундамент» его AQA-подготовки, 15 разделов) и просит новый раздел
с уроками по нему; пробелы — доресерчить; затем задеплоить. Ресерч: RS-01 (нарезка:
P0 7 уроков / P1 4 / P2 справочник), RS-02 (9 неточностей md; 6 senior-гэпов: GIL,
refcount+GC, generator frames, closure cells, дескрипторы, small-int cache; банк 32 URL),
RS-03 (перепрогон спайков на python3.12; артефакты — evidence/spikes/ + run-log-py312.txt),
Explore (мультитрек: бэк готов, фронту нужна малая доработка), RS-baseline.

## Класс результата
Контентная волна software-UI продукта (lesson-as-data: research-контент + код-данные +
малая UI-доработка мультитрека). Improvement поверх verified-продукта.

## Целевая планка
ГОТОВЫЙ ПРОДУКТ (ступень ≥3): уроки на планке существующего трека (не черновики),
в проде, в петле FSRS. Движок/дизайн/методика — LOCKED, не пересматриваются.

## Язык продукта
`UI language: ru`. Allowlist: код/идентификаторы/термины EN (Python, yield, GIL, dis…),
дословные EN-цитаты первоисточников.
| Понятие | Канон | Запрещено |
|---|---|---|
| имя → объект | «имя ссылается на объект» | «переменная хранит значение» (в Python-треке) |
| изменяемый/неизменяемый | mutable / immutable (EN в коде, рус. в прозе) | «константный» |
| байткод | байткод (панель `dis`) | «IL» (это C#-трек) |
| замыкание | замыкание (cell) | «лямбда-захват» |
| поток повторов | Серия / Закреплено / XP (как в продукте) | Стрик/Освоено/Опыт |

## Ставка вау
Ось ступени 4: **«сеньор видит невидимое»** — под-капотные сущности CPython, которых нет
в обычных курсах, показаны анимацией: ячейки замыканий (`__closure__`), замороженный кадр
генератора, слои декораторов, байткод-панель `dis` (переиспользование IL-панели), event
loop, GIL. | Ставочные фичи: уроки 6 механизмов + панель dis + собес-флоу md как фича
(«на собесе спросят → отвечать так» = misconceptions/карточки) с сеньорской глубиной. |
Риск скуки: справочные разделы md (строки/коллекции-API/stdlib) → контрход: в волне они
только уроки-шпаргалки с сюжетными сегментами, ядро волны — механизмы с сюжетом.

## Скоуп
### P0 (без этого провал) — ВЫБОР ГЕЙТА: волна 1 = ПОЛНЫЕ 11 уроков (бывш. P0+P1)
- Уроки волны (нарезка RS-01):
  `py-names-objects` (модель памяти: имена→объекты, mutable/immutable, mutable default,
  small-int cache/interning, refcount+GC) · `py-closures-scope` (LEGB, cells, late binding) ·
  `py-decorators` (слои, functools.wraps, @pytest.fixture-связка) ·
  `py-generators` (yield, замороженный кадр, setup/teardown фикстур) ·
  `py-context-managers` (__enter__/__exit__, contextlib, with) ·
  `py-async-await` (event loop, корутина-как-генератор, GIL) ·
  `py-object-model` (классы, MRO, дескрипторы→bound methods, property) ·
  `py-type-hints` (+Pydantic preview) · `py-exceptions` (иерархия от OSError — фикс md) ·
  `py-collections-hash` (dict под капотом) · `py-args-unpacking`.
- Каждый урок: сегменты только через `at`; ≥4 анимированных разбора (флагманы ≥6);
  каждое утверждение — первоисточник (банк RS-02), цитаты дословно; 9 неточностей md
  НЕ воспроизводить (исправленные формулировки по RS-02); карточки predict-output
  с `verify.expect` = реальный stdout **python3.12** (авторский гейт G-EXEC-PY).
- Мультитрек-UI (малая доработка, generic): группировка home по track + лейблы треков
  (убрать хардкод «Ядро C#»), поле `lang` в сегменте + подсветка Python в hlcode.ts,
  бейдж панели байткода по языку («IL»/«dis»), seed нового трека (track id T7-Python
  или следующий свободный — фиксируется в design.md).
- Деплой: commit+push в main → CI (test+build+viz-fit) зелёный → образ → VPS.
### P1 (важно)
- Мисконцепции «на собесе спросят» из md → misconceptions/карточки уроков (РЕШЕНИЕ ГЕЙТА:
  собес-флоу сохраняем как фичу + сеньорский взгляд: каждый собес-ответ подкреплён
  под-капотным механизмом, не зубрёжкой).
- СМЕЖНОСТИ md (просьба пользователя): хребет §0 («карта всех блоков», таблица «механизм →
  где встречаешь: @pytest.fixture/@allure.step, yield-фикстуры, with httpx/testcontainers,
  AsyncClient/Locust») сохраняется в уроках — у каждого урока сегмент/takeaway «где
  встречаешь в своих инструментах» + вводный сегмент-карта в первом уроке трека
  («из чего сделаны твои инструменты — 6 механизмов»). Приём md «забыл — выведи из
  соседнего блока» → в explain/okText карточек, где естественно.
### P2 — РЕШЕНИЕ ГЕЙТА: ВКЛЮЧЕНО В ВОЛНУ («делай все включительно даже p2»)
- Справочник (§2 строки, §4 поток, §13 stdlib, §14 pathlib, §15 idioms) как уроки-шпаргалки
  СУЩЕСТВУЮЩЕГО формата: 2–3 анимированных сегмента там, где есть сюжет (срезы, файл/pathlib,
  idioms-паттерны) + карточная колода predict-output; G3-порог для шпаргалок = стандартный
  (≥2–3 сегмента), G2/G2a — в полный рост.
- Track-уровневая статистика прогресса — если требует замен экранов против LOCKED, то
  ограничиться группировкой per-lesson по трекам (не блокер волны, честно доложить).
- GIL: глубокие сегменты в py-async-await; отдельный урок free-threading 3.13 — решение
  design-фазы по объёму (не обещан).

## Не входит в скоуп

  Python-уроках нет врезок «как это в C#».
- Изменения движка сверх generic-доработок выше; редизайн; методика/FSRS; бэкенд-раннер
  Python (`/api/run-python` НЕ строим — эталоны считает автор локальным python3.12,

- Остальные блоки конспекта пользователя (1–11: pytest, Allure, API…) — будущие волны.

## Риски и допущения
- Конфликт аудиторий: РЕШЕНО ГЕЙТОМ — собес-флоу md сохраняется как фича, тон и глубина
  сеньорские (каждый собес-ответ обоснован механизмом под капотом).
- Версии Python: эталоны и все факты версии — ТОЛЬКО python3.12 (RS-03: перепрогон 26
  спайков, 2 расхождения с 3.9 — PEP 604 и опкоды dis; тексты ошибок — из 3.12).
- Исполнение Python: `/api/run-python` НЕ строим — эталоны считает автор локальным
  python3.12 при авторинге (зафиксировать ADR-0001 в фазе 3).
- Объём: волна = 11 уроков (выбор гейта) ≈ 6–9 билдер-прогонов + UI-прогон; критик
  предупредил о риске G2/G3 при таком объёме → чекпойнт после 1-го урока обязателен,
  качество не срезается ради скорости (урок не сдаётся при нарушении G2/G3).
- FSRS-дозирование: seed новых уроков вводит карточки как new → в day-очередь попадают
  постепенно (как при вводе C#-волны); проверка на VERIFY: due-очередь дня после сида
  не взрывается (число new/день ограничено поведением очереди — замерить и записать).
- Пейсинг: у каждого урока — estMinutes из числа сегментов (калибровка: boxing 7 ≈ 9 мин);
  сессия остаётся короткой, флагманы ≤ 10–12 мин.
- P2 «шпаргалки-справочник»: если делаем позже — ТОЛЬКО карточками существующего формата
  (никаких новых экранов против LOCKED без отдельного гейта).
- Деплой-джоб может быть в graceful-skip (если VPS-секреты не заданы) — тогда сдаём
  «CI зелёный + образ в GHCR» и сообщаем пользователю шаг активации.
