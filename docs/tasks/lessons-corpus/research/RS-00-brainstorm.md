# RS-00 — Брейншторм (≥10 углов, серийно) + синтез critic-брейнштормеров

North Star: полный корпус глубоких анимированных уроков (книги + пересобранный трек C# + инструменты бэкенда) в существующем TMA с петлёй FSRS.

## Углы оркестратора (серийная генерация, каждый ≠ предыдущим)

1. **Сеньор-учащийся (целевая персона).** Ценность — инсайт «как я бы сам это спроектировал», не конспект. → ТЗ: гейт качества «урок даёт механизм + why», запрет пересказа-выжимки.
2. **Скептик «зачем, если книга есть».** Урок обязан быть лучше главы: короче, анимирован, встроен в петлю повторов, самодостаточен. → ТЗ: каждый урок несёт карточки FSRS и работает без книги.
3. **Юзер в худших условиях (метро, 5 минут, телефон).** Урок дозирован, автоиграет, не требует IDE. → ТЗ: соблюдать лимиты длины/пейсинга существующего движка.
4. **Владелец продукта.** Сотни уроков без иерархии — свалка; нужна навигация трек→раздел→урок и волны выкатки. → ТЗ: структура разделов и порядок волн — часть деливерабла, не «потом».
5. **Методист.** Порядок глав книги ≠ порядок обучения; пререквизиты между книгами (Грокаем → Кормен; C#-трек → Клири). → ТЗ: явный learning path / зависимость разделов.
6. **Инженер движка/бюджета.** 20 книг × разделы + полный C# + инструменты = сотни уроков; каждый — TS-файл с анимациями. Цена урока измерима пробой. → ТЗ/гейт: честная цена планки, волны по приоритету, никакого «всё сразу».
7. **Юрист.** Книги под копирайтом: уроки — оригинальный пересказ идей своими словами, без дословных кусков текста/рисунков; цитаты — короткие, с атрибуцией. → ТЗ: политика производного контента.
8. **Конкурент (Anki/курсы).** Сигнатура продукта — живая диаграмма + FSRS; уроки обязаны использовать анимации по-настоящему, не как слайды. → ТЗ: viz-fit каждой темы.
9. **Редактор дублей.** Пересечения: 4 книги алгоритмов; 2 микросервисные; 3 книги C#-внутренностей ∩ трек C# по докам; DDIA ∩ Таненбаум ∩ Бёрнс. → ТЗ: дедупликация тем, взаимные ссылки, одна тема — один канонический урок + перспективы книг.
10. **Существующий пользователь (py-трек).** Удаление C#-уроков не должно ломать py-трек, очередь FSRS и прогресс в БД. → ТЗ: план миграции/чистки сида и БД, py-* неприкосновенен.
11. **Инвестор/волны.** Первая волна должна дать ощутимую ценность (новый C#-трек + топ-книги), хвост — потом. → гейт: пользователь ратифицирует приоритет волн.
12. **Слабое зрение/доступность.** Текст в анимациях мелкий на телефоне — следовать нормам движка (типографика, контраст), не изобретать. → ТЗ: наследуем существующие нормы.

## Дайджест critic-максималиста
(вложен ниже, синтез в «Следствия»)

1. Concept-DAG вместо «книга=трек»: единица — концепт, тянущий 2-4 книги; coverage-матрица книга→концепт.
2. Единый виз-язык: расширить примитивы движка до distributed-словаря (реплика, партиция, clock skew).
3. «Машинная панель» generic: IL / wire-байты / план запроса / счётчики GC — уровень ниже абстракции книги.
4. Executable ground truth: карты верифицируются реальным замером, не мнением LLM.
5. «Книга vs рантайм 2026»: для книг <2018 обязательный сегмент актуализации, факты сверяются с Learn/замером.
6. Кейс-стержень: одна сквозная система через модули (ломается, шардируется, профилируется).
7. Жанр «почему так спроектировано»: trade-off как анимация умирающих альтернатив.
8. Predict-scene карты: стоп перед ключевым кадром, ответ — следующее состояние диаграммы.
9. FSRS-interleaving контраст-парами мисконцепций.
10. Mastery-gate порядка обучения по стабильности FSRS.
11. Refutation-первый экран: pre-test «сеньор вроде знает».
12. Сегмент «книги спорят»: контроверза с цитатами и страницами.

## Дайджест critic-пессимиста

1. Масштаб ~300–600 уроков (20 книг × 15–30 концептов; сейчас 21 урок ≈ 6200 строк) → якорь объёма в ТЗ, транши с гейтами вместо «полного корпуса».
2. **[ЧУВСТВИТЕЛЬНО → анкета гейта]** Копирайт: PDF — коммерческие переводы; дословные цитаты из них в раздаваемом продукте — двойное нарушение → политика: пересказ механизмов, цитаты только из открытых доков/спек; решение пользователя.
3. **[ЧУВСТВИТЕЛЬНО → анкета гейта]** «Все уроки» нефальсифицируемо → полнота = concept-DAG с покрытием X% + cut-list.
4. Удаление C#-уроков → сироты review_state/progress_events в SQLite, «призрачные» due → миграция/tombstone + харнесс «due ⊆ зарегистрированных».
5. Дубли (4 алгоритмические, 3 микросервисные, 2 async, Рихтер-2013 vs доки) → concept-DAG «концепт → один владелец»; Рихтер — история, истина = learn.microsoft.com.
6. Книги-советы (Прагматик, Чистая архитектура, DDD) не ложатся на exec-истину → отдельный формат/гейт для non-exec доменов, порог «0 буллет-конспектов».
7. Читаемость: .djvu Седжвик, возможные сканы без текстового слоя; OCR врёт в числах → инвентаризация ДО планирования, нечитаемое = явный срез.
8. Конвейерная деградация качества → выборочный аудит точности на каждый транш + метрика плотности анимаций (не только геометрический viz-fit).
9. UX/нагрузка: сид всего корпуса взорвёт due в день 1 → unlock по prereqs, лимит новых карт/день, навигация треки→модули как фича.
10. Бандл: 500 уроков в одном Vite-бандле ≈ 10–20 МБ в WebView → code-splitting + инкрементальная верификация до массового авторинга.
11. Цена срезов для cut-list: срез книги = потерянная глубина темы; срез миграции = потерянное расписание повторов.

## Следствия для ТЗ (синтез)

- Структурная единица корпуса — **концепт/раздел, не книга**: concept-DAG + coverage-матрица (книга→концепт, док→концепт), «один концепт — один канонический урок», книги дают перспективы/контроверзы.
- **Объём и волны — решение пользователя на гейте**: честная цена (уроков на майлстоун из пробы), транши: W1 = трек C# заново + core-инструменты + топ-книги; хвост — волнами.
- **Политика источников/копирайта — анкета гейта**: пересказ механизмов своими словами; дословные цитаты — только открытые доки/спеки; переводы-PDF = источник структуры и идей, не текста.
- **Деструктив**: удаление C#-уроков — с миграцией сида/БД (tombstone), py-трек неприкосновенен; преавторизация в ТЗ.
- **Два формата уроков**: exec-домены (C#, инструменты, алгоритмы) — с исполняемой истиной; non-exec (архитектура/советы) — сценарные симуляции/контроверзы, гейт «0 конспектов».
- Инфраструктурные пререквизиты массового авторинга: code-splitting, лимит новых карт/день, навигация по модулям, инкрементальная верификация — кандидаты в фиче-лист ДО волны книг.
- Ставка вау: машинная панель под абстракцией книги, «книга vs рантайм 2026», predict-scene, контраст-пары FSRS — отобрать 2-3 в паспорт вкуса, не всё сразу.
