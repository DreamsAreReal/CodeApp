# RS-02a. Инвентарь книг (часть 1/2): 10 книг из books/develop/

Дата: 2026-07-17. Метод: чтение первых 20–25 страниц каждого PDF (текстовый слой через
pdfplumber; для скана Кормена — рендер страниц оглавления в PNG через pypdfium2
и чтение глазами). Артефакты извлечения: scratchpad `books_info.json`,
`books_info_extra.json`, `cormen_p5.png`–`cormen_p10.png`.

## 1. Вопросы

Для каждой книги: (a) точное название/автор/год издания; (b) структура частей/глав;
(c) 5–12 крупных разделов-кандидатов на разделы уроков; (d) оценка числа уроков
(1 урок ≈ 1 глубокая тема главы); (e) читаемость PDF (текстовый слой).

## 2. Корпус и покрытие

Корпус — закрытый: 10 файлов из ТЗ, все найдены на диске, все 10 обследованы (100%).
Два файла в ТЗ названы с ASCII-апострофом `'`, реальные имена содержат U+2019 `'`:

- `Ньюмен Сэм - Создание микросервисов, 2-е издание (Бестселлеры O'Reilly) - 2023.pdf` (апостроф U+2019)
- `Бёрнс Б. - Распределенные системы. Паттерны проектирования (Бестселлеры O'Reilly) - 2019.pdf` (апостроф U+2019)

Это важно для любого кода конвейера, который будет открывать файлы по имени.

Источник всех фактов — сами PDF (выходные данные, «Краткое содержание», «Оглавление»).
Уверенность по названиям/структуре — высокая (первичка). Оценка числа уроков —
экспертная экстраполяция из оглавлений, уверенность средняя.

### Книга 1. Ньюмен — Создание микросервисов

- **Точное название**: Сэм Ньюмен. «Создание микросервисов», 2-е изд. — СПб.: Питер,
  2023. — 624 с. (серия «Бестселлеры O'Reilly»), ISBN 978-5-4461-1145-9.
  Оригинал: Sam Newman, Building Microservices, 2nd ed.
- **Структура**: части («Часть I. Основы» видна в кратком содержании), главы:
  гл. 1 «Что такое микросервисы», гл. 2 «Как моделировать микросервисы» (DDD,
  ограниченный контекст, Event Storming), гл. 3 «Разделение монолита на части»,
  гл. 4 «Стили взаимодействия микросервисов»; далее по оглавлению видны: межпроцессное
  взаимодействие (GraphQL, брокеры сообщений, форматы сериализации), гл. 6 «Рабочий
  поток» (ACID, распределённые транзакции, двухфазная фиксация), развёртывание
  (физические/виртуальные машины, контейнеры), тестирование (сквозные тесты),
  безопасность (авторизация, JWT), гл. 13 «Масштабирование» (четыре оси), декомпозиция
  UI (по страницам, по виджетам), организационные темы (доверенные коммиттеры,
  технический долг, шаблон микросервиса).
- **Кандидаты-разделы уроков** (8): 1) что такое микросервисы и когда они не нужны;
  2) моделирование границ сервисов через DDD; 3) стратегии разделения монолита;
  4) стили и технологии межсервисного взаимодействия; 5) распределённые транзакции
  и саги; 6) развёртывание и контейнеры; 7) тестирование микросервисов; 8) безопасность,
  масштабирование и организационные аспекты.

- **Читаемость**: текстовый слой ЕСТЬ, извлекается чисто.

- **Точное название**: Т. Кормен, Ч. Лейзерсон, Р. Ривест, К. Штайн. «Алгоритмы.
  Построение и анализ» (CLRS), русское издание 2013 г., 1324 с. в PDF.
- **Структура** (оглавление прочитано глазами с рендера страниц 5–6): 8 частей,

  и властвуй + вероятностный анализ; 3) сортировки и порядковые статистики;
  4) хеш-таблицы; 5) сбалансированные деревья (BST, красно-чёрные, B-деревья);
  6) динамическое программирование; 7) жадные алгоритмы и амортизационный анализ;
  8) графы: обходы, MST, кратчайшие пути; 9) потоки в сетях; 10) NP-полнота
  и приближённые алгоритмы.
- **Оценка уроков**: 40–60 (самая ёмкая книга; для сеньорского корпуса разумна
  выборка, а не полное покрытие).
- **Читаемость**: текстового слоя НЕТ — скан (0 символов на пробах стр. 6, 9, 13,
  101, 501, 1001). Для конвейера уроков потребуется OCR либо ручная работа
  с рендерами страниц. БЛОКЕР для автоматической выжимки текста.

## 8. Рекомендация

1. Суммарная ёмкость корпуса (часть 1/2): **~190–260 уроков**, реалистичная рабочая
   оценка **~220**.
2. Приоритет для сеньорского корпуса: Клеппман, Ньюмен+Ричардсон (парой), Нейгард,
   Таненбаум — максимальная плотность «глубоких тем на главу» и текстовый слой чистый.
3. «Грокаем алгоритмы» не заводить как отдельный трек — использовать как
   вспомогательный источник интуиции внутри уроков по Кормену (полное пересечение тем).
4. Кормен — единственный БЛОКЕР по читаемости (скан без текстового слоя, 164 МБ):
   либо OCR-прогон (tesseract+rus), либо построчная работа с рендерами страниц,
   либо перенести в часть 2 скоупа. Решение — за оркестратором.
5. В коде конвейера использовать имена файлов с U+2019 (см. п. 2) или matching по glob.
