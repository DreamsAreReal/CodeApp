# Фиче-лист — lessons-corpus, волна 1

North Star: корпус глубоких анимированных уроков в существующем TMA с FSRS; каждый урок показывает уровень ниже абстракции источника.

## Дисциплина статусов
- Каждая фича рождается `todo`. Порядок священен: F1 — walking skeleton.
- `self-pass` ставит builder только с доказательством (команда → вывод / скриншоты в `docs/tasks/lessons-corpus/evidence/F<N>/`).
- `verified` проставляет оркестратор по вердикту evaluator-а. `cut` — только решением по протоколу, фича режется целиком.
- Смоук новой сессии builder-а: `npm run build` + до F5 — явный прогон 5 харнессов (см. design «Инструментарий»); после F5 — `npm run verify:all -- --lessons <скелет+золотой путь>`.
- Уроки: фича = 3–4 урока; критерий «урок готов» ЕДИНЫЙ (см. DoD ниже), в приёмке фичи не повторяется.

**DoD урока (Definition of Done, применяется к каждому уроку волны)**: файл `app/src/lessons/cs/<slug>.ts` + регистрация в registry + сид-JSON; ≥4 сегмента с динамикой узлов, ≥2 exec-карточки (stdout снят реально через run-csharp); **машинная панель с реально снятым числом/артефактом** (IL-строки из реальной компиляции, замер аллокаций/GC-счётчик, вывод команды — команда и сырой вывод сохраняются в evidence фичи; декоративная панель без замера = не DoD); sources[] = URL из RS-03 v2 (живые); инкрементальный verify (E2E+viz-fit+density по уроку) зелёный; тон/плотность — по эталону boxing.ts.
