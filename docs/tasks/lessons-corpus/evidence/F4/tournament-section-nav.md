# F4 · Турнир: нулевой экран — навигация трек → раздел → урок

Дата: 2026-07-18. Судья: builder (единственный пишущий агент; парные сравнения в обоих порядках).
Скриншоты: `{375,768}-variant-{A,B,C}.png` (fullPage, dev-user, онбординг пропущен).

## Три радикально разных варианта
- **A — MINIMAL**: тихий uppercase-разделитель раздела («ТИПОВАЯ СИСТЕМА») + тонкая
  подпись прогресса («0 из 3 пройдено»), дальше плоские карточки уроков. Раздел = метка,
  не контейнер. Наименьший визуальный вес.
- **B — BOLD**: раздел = КАРТА-заголовок (иконка-плитка + название + «N из M пройдено» +
  кольцо прогресса %), уроки вложены внутрь на кремовой подложке. Рекомендованный урок —
  коралловая рамка + пилюля «ПРОДОЛЖИТЬ ЗДЕСЬ». Раздел читается как глава, по которой идёшь.
- **C — RAIL**: слева ступенчатый рельс с номером раздела (кораловый «1»), справа — заголовок
  раздела и раскрытые уроки. Референс-стиль (лестница), структура-forward.

## Парные сравнения (оба порядка)
| Пара | Порядок 1 | Порядок 2 | Победитель | Обоснование |
|---|---|---|---|---|
| A vs B | B > A | B > A | **B** | Кольцо прогресса раздела даёт мгновенный статус «где я в главе»; контейнер-карта — сильнее иерархия. A чище, но раздел = плоская метка, слабее ощущение «движения по главе». |
| B vs C | B > C | B > C | **B** | Рельс C элегантен для МНОГИХ разделов, но в волне 1 у CS ровно ОДИН раздел — рельс схлопывается в отступ, преимущество не проявляется. Карта B работает при любом числе разделов; кольцо информативнее номера. |
| A vs C | A > C | A > C | **A** | Оба слабы при 1 разделе; A чище (нет полу-пустого рельса), C обещает структуру, которую пока не может показать. |

## Итог
**Победитель — B (BOLD section-card + progress ring).** Причины:
1. North Star «как будто сам спроектировал»: раздел-глава с явным прогрессом — «иду по главе».
2. Масштабируется на N разделов (S1→S7→S17→S18→S2) без деградации (в отличие от рельса C при 1 разделе).
3. Кольцо прогресса раздела закрывает требование design «Опыт» (прогресс раздела виден).
4. Держит планку на 375 И 768 (0 обрезок, см. 768-variant-B.png).

Интегрирован как `F4_WINNER = "B"` в `app/src/app/home.ts` (дефолт `sectionNavBody`).
Проигравшие A/C остаются переключаемыми через `window.__f4Variant` только для этого турнира
(не в проде — дефолт всегда B).
