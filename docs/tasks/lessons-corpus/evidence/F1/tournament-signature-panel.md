# Турнир signature-панели — CS.S1.value-types-copy (F1)

Правило (brief §Турниры): 3 радикально разных черновика подачи «машинной панели»,
парные сравнения в ОБОИХ порядках, победитель интегрируется в урок. Ось судейства —
North Star: «уровень НИЖЕ абстракции источника» + паспорт вкуса: «кульминация урока —
панель машинного уровня с РЕАЛЬНО снятым числом», signature = «вся смелость туда».
Каждый черновик стоит на реальном артефакте (evidence/F1/value-types-copy-exec-il.txt).

## Черновики

**A — IL-лента реальной компиляции.**
Панель = IL структ-копии, синхронная с кадрами: `ldarg.0 / stloc.0` (копия структуры
целиком в свежий локал — нет `newobj`, нет кучи), затем `ldloca.s 0 / stfld` (мутация только
копии) и `ldarg.0 / ldfld` (оригинал не тронут). Артефакт: настоящий ilspycmd Release-IL.
- Плюс: буквальный машинный слой — «отпечаток копии» в байткоде; ровно жанр boxing.ts (IL-панель).
- Минус: IL структ-копии почти пустой (`ldarg/stloc`) — «где само число?» слабее, чем у B.
  Копирование виден как ОТСУТСТВИЕ опкода (нет box/newobj) — сильная идея, но требует
  «читать пустоту».

**B — счётчик-замер аллокаций.**
Панель = живой счётчик `GC.GetAllocatedBytesForCurrentThread()`: структ-копия = **0 байт**,
`new class` = **24 байта** (реальный замер). Кульминация — цифра 0 vs 24 на табло.
- Плюс: РЕАЛЬНО СНЯТОЕ ЧИСЛО в чистом виде (планка паспорта — «панель с реально снятым числом»);
  0 vs 24 моментально бьёт в «value types не трогают кучу» — это и есть уровень ниже абстракции.
- Плюс: контраст-пара встроена (struct 0 / class 24) — визуально самодоказуемо.
- Минус: замер — про давление на кучу, а не про САМУ семантику копии (что копируется). Один
  урок «копирование» — число про аллокацию слегка вбок; но struct-vs-class копия ровно про это.

**C — X-ray стек/куча.**
Панель = рентген памяти: поля struct inline на стеке (X,Y соседними слотами), рядом — полная
копия; у class переменная = указатель в кучу. + случай shared-reference (TaggedInteger):
Number скопирован, List — общий.
- Плюс: показывает ЧТО копируется (весь value inline) vs ЧТО шарится (ref-член) — прямо в точку
  темы «семантика копирования».
- Минус: это язык обычных сегментов урока (stack/heap-диаграммы) — как SIGNATURE слабее: «ещё
  одна диаграмма», а не машинное табло. Смелости меньше (паспорт: signature = вся смелость туда).

## Парные сравнения (оба порядка)

Критерий выбора в каждой паре: (1) реально снятое число на табло; (2) «уровень ниже абстракции»;
(3) смелость/несводимость к обычному сегменту; (4) моментальность «вау».

- **A vs B** → B. У B живое число 0 vs 24 (кульминация-табло); IL структ-копии почти пустой,
  «вау» слабее. Уровень ниже — у обоих, но у B он ещё и measured.
- **B vs A** (обратный) → B. Тот же исход: измеренная цифра > почти-пустой IL как кульминация.
- **A vs C** → A. IL — машинный слой и смелее (табло байткода); C — обычная диаграмма памяти.
- **C vs A** (обратный) → A. Тот же исход.
- **B vs C** → B. B несёт measured-число и контраст-пару 0/24; C — язык обычных сегментов.
- **C vs B** (обратный) → B. Тот же исход.

Итог: **B > A > C**, устойчиво в обоих порядках каждой пары.

## Победитель: B — счётчик-замер аллокаций (0 vs 24 байта)

Причина: точно попадает в планку паспорта («панель машинного уровня с РЕАЛЬНО снятым числом»,
signature = вся смелость), даёт моментальный контраст 0/24, и это буквально «уровень ниже
абстракции» темы копирования (что копирование value-типа НЕ идёт в кучу).

## Интеграция (как в уроке)
- Signature-сегмент s5 «Машинная панель · счётчик аллокаций»: два прогона на одном табло —
  `struct s2 = s1` → счётчик 0 байт; `new PointC` → счётчик 24 байта; кадры двигают числа на
  панели (динамика узлов). Число — из EXEC 2 (реальный замер), не выдумано.
- IL-идея (A) НЕ выброшена: живёт отдельным сегментом s4 (IL структ-копии `ldarg/stloc`) —
  проигравший турнир черновик становится обычным сегментом, но SIGNATURE-табло = B.
- X-ray/shared-ref (C) → сегменты s1–s3 и s6 (layout inline + общий ref-член TaggedInteger).
