# ТЗ — lessons-corpus: корпус уроков (книги + трек C# заново + инструменты бэкенда)

## Цель
В приложении (TMA «уроки-как-данные» + FSRS) существует расширенный корпус уроков: пересобранный с нуля по официальным докам трек C# (без 6 старых уроков, с миграцией), трек «Инструменты C#-бэкенда» и уроки по разделам всех читаемых книг из `books/develop/` — волнами, каждая волна проходит все харнессы зелёными и аудит плотности.

## Контекст
Сейчас 19 уроков (6 C#, 13 py), 59 карточек, все харнессы зелёные (RS-baseline v2). Цена авторинга: 6–10 exec-уроков/майлстоун (проба, `.spikes/probe/REPORT.md`). Сводный объём корпуса (потолок; оценка «C# 15–20 уроков» в RS-04 УСТАРЕЛА, истина — RS-03 v2):

| Источник | Мин | Макс | Волны |
|---|---|---|---|
| Трек C# (доки, RS-03 v2) | 108 (core) | 141 (+optional) | W1 (~41) → W2+ |
| Инструменты (RS-04) | ~60 (CORE+нач. INFRA) | ~130 | W2+ |
| Книги читаемые, 16 шт. (RS-02a/b; без Кормена) | ~240 | ~320 | W3+ (с дедупом может ужаться) |
| Книги после OCR: Кормен, Фаулер, EIP | ~70 | ~100 | P2 |
| **Итого** | **~480** | **~690** | ~60–100 майлстоунов |

Ключевые инсайты: единица корпуса — концепт, не книга (дубли: Рихтер/Кокоса/Дэвис ⊂ C#-трек; Ньюмен↔Ричардсон; Клеппман↔Таненбаум); 4 книги нечитаемы без OCR (Кормен, Фаулер, EIP, Седжвик); non-exec темам сегодня доступны только MCQ-карточки; лимита новых карт/день нет (due дня 1 = весь каталог) — блокер массового наполнения. RS-00…RS-04, R-02.

## Класс результата
mixed: document(информировать)-контент в форме кода-данных внутри существующего продукта; режим improvement.

## Целевая планка
ГОТОВЫЙ ПРОДУКТ (ступень ≥3) **на уровне каждой волны**: волна = завершённый, выкатываемый прирост (все уроки волны на планке эталонов, харнессы зелёные, навигация не деградирует). Полный корпус — дорожная карта волн; статусы DONE-WAVE-N между волнами.

## Ставка вау
Ось ступени 4: **«уровень ниже абстракции источника»** — каждый урок показывает машинный слой под темой (IL/стейт-машина/бакеты/аллокации для C#; план запроса/wire-формат/ack-механика для инструментов; для книг — сегмент «книга vs рантайм 2026»: утверждение книги сверено живым замером или Learn). | Ставочные фичи: машинная панель + exec-карточки с реальным stdout. | Риск скуки: конвейерная одинаковость сегментов при массовом авторинге → контрход: обязательная динамика узлов в каждом сегменте (G7) + контраст-пары карточек (мисконцепция vs истина) + выборочный аудит каждый майлстоун.

## Паспорт вкуса
**Референс-эталон**: собственные флагманы — `boxing.ts` (7 сегментов, IL-панель, 188 строк) и `py-generators.ts` (6 сегментов, реальный замер 38.4 MB vs 200 B) — деконструкция в RS-01/RS-baseline. Структура голоса: сеньорский тон без азов; сегмент = механизм в движении (узлы меняются между кадрами), не иллюстрация; кульминация урока — панель машинного уровня с реально снятым числом; карточка = предсказание вывода с детерминированной проверкой.
**Направление**: наследуем frozen-дизайн продукта (палитра #F6F1E9/#F0533A, Rubik/Onest/JetBrains Mono, примитивы slot/ref/obj/chip/gate, auto-layout v2) — новых визуальных систем не изобретать.
**Анти-слоп запреты (жанровые)**: 0 сегментов «chip + подпись» без динамики узлов; 0 буллет-конспектов глав; 0 карточек с verify.run, содержащим ответ (echo-фальсификация); запрет суперлативов в объяснениях («мощный», «элегантный»); запрет «Feature 1/2/3»-структур; факты рантайма из книг <2018 — только со сверкой (Learn или замер).
**Signature-элемент**: машинная панель «уровнем ниже» в каждом уроке (IL / бакеты / стейт-машина / план запроса / wire) — вся смелость туда.

## Риски и допущения
- Конвейерная деградация качества при массовом авторинге → G7-аудит каждый майлстоун + ротация builder-а по волнам.
- OCR-книги (Кормен и др.) двигаются в P2; допущение: пользователь согласен (иначе меняется состав волн).
- Non-exec домены: если design не даст нового verify-механизма — карточки MCQ (легально по types.ts); допущение фиксируется ADR.
- Оценка 6–10 уроков/майлстоун может не подтвердиться на волне → бюджетный чекпойнт после M1 волны 1.
