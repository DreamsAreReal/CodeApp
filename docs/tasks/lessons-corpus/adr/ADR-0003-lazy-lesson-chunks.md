# ADR-0003 — перевести уроки на dynamic import (чанк на урок) через registry.ts

## Статус
Принято 2026-07-18

## Контекст
Все 19 уроков статически импортированы в `index.ts` → один бандл 571.88 KB raw / 141.75 KB gzip (замер `npm run build`, RS-baseline v2). При 100+ уроках начальный чанк растёт линейно — в Telegram WebView это блокер (brief P0, G1: начальный чанк ≤ базлайн +10%).

## Рассмотренные варианты
| Вариант | Плюсы | Минусы | Замеры из спайка |
|---|---|---|---|
| A. `import()` на урок в манифесте (`registry.ts`), Vite auto-chunking | штатный механизм Vite (без новых зависимостей); чанк на урок; метаданные (заголовок/карты) остаются в начальном чанке | харнессы, читающие LESSONS, ждут async-загрузку | базлайн 571.88 KB raw — точка отсчёта; приёмка F1: вывод `npm run build` показывает отдельные чанки уроков и начальный чанк ≤ базлайн +10% |
| B. Оставить статику, резать позже | ничего не делать сейчас | каждый урок волны 1 увеличивает начальный чанк; переделка на 60 уроках дороже | — |

## Решение
Вариант A — до массового авторинга (порядок фич: F1-скелет уже на registry). Новых зависимостей нет (dynamic import — API Vite/ES), поэтому спайк-обязанность закрывается исполняемой приёмкой F1 (замер чанков в выводе build) — фиксация здесь, до кода.

## Последствия
- Харнессы (`run.mjs`, `viz-fit.mjs`, `new-lessons.mjs`) переводятся на загрузку через registry (await load()).
- Принятые риски: первая открытие урока = сетевая догрузка чанка (~единицы KB gzip) — приемлемо для TMA single-origin.
